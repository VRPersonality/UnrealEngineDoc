13/**
14 * UBasic2DLineSetComponent and UBasic3DLineSetComponent are components that draw a set of lines as small squares. The 2D version stores XY coordinates with
15 * Z set to 0 whereas the 3D version stores XYZ coordinates.
16 * All lines have the same color, size, and depth bias based on what is set via SetLineSetParameters(). The lines are drawn as two triangles, i.e. a
17 * square, orthogonal to the view direction. The actual line size is calculated in the shader using a custom material.
18 *
19 * Before inserting any lines via AddElement(), the maximum number of lines that will be added needs to be declared via ReserveElements().
20 * The actual number of lines that were added is available via NumElements().
21 *
22 * To support both 2D and 3D with minimal code duplication, the line set components are composed from the UBasicLineSetComponentBase class for the component
23 * specific functionality, and an instantiation of the TBasicElementSet template class for either FVector2f or FVector3f that encapsulates all line specific
24 * functionality. The final derived classes UBasic2DLineSetComponent and UBasic3DLineSetComponent contain identical boilerplate code to facilitate calls that
25 * need to go both to the component base class and the line base class, e.g. Clear() will both mark the component as dirty and delete all lines.
26 */

58 /** Update component bounds using a provided function that determines the box around all line positions.
59 * This unifies how the bounds are calculated while keeping the calculation of the box around all line positions generic. */
60 template <typename CalcLineBoxFn>
61 FBoxSphereBounds UpdateComponentBounds(const FTransform& LocalToWorld, bool bLinesDirty, CalcLineBoxFn&& CalcLineBox) const
62 {
63 bBoundsDirty |= bLinesDirty;
64 if (bBoundsDirty)
65 {
66 // Calculating an FBox around a set of lines is specific to the line type, and thus we call out to a provided type specific lambda.
67 const FBox Box = CalcLineBox();
68
69 Bounds = FBoxSphereBounds(Box);
70 bBoundsDirty = false;
71
72 // TODO: This next bit is not ideal because the line size is specified in onscreen pixels,
73 // so the true amount by which we would need to expand bounds depends on camera location, FOV, etc.
74 // We mainly do this as a hack against a problem in orthographic viewports, which cull small items
75 // based on their bounds, and a set consisting of a single line will always be culled due
76 // to having 0-sized bounds. It's worth noting that when zooming out sufficiently far, the
77 // line will still be culled even with this hack, however.
78 // The proper solution is to be able to opt out of the orthographic culling behavior, which is something
79 // we need to add.
80 if (Box.IsValid)
81 {
82 Bounds = Bounds.ExpandBy(Size);
83 }
84 }
85 return Bounds.TransformBy(LocalToWorld);
86 }
