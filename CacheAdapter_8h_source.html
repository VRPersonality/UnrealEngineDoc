31 /**
32 * Base adapter type for interfacing with Chaos simulation caches. Any component wishing to use caches must have a
33 * compatible record adapter and a playback adapter to produce and consume cache data.
34 *
35 * This base interface should not be used to implement those adapters however but the derived classes
36 * FCacheRecordAdapter and FCachePlaybackAdapter (declared below) should instead be used.
37 */

45 Derived

56 /**
57 * Query whether InComponentClass can be driven by this adapter. If a class implementing this interface says it
58 * can support a component it *may* be chosen as the driving adapter for that component and will be expeted to
59 * be able to handle Pull/Push operations if it is chosen.
60 *
61 * If the adapter returns SupportType::Derived and another adapter returns SupportType::Direct then the adapter
62 * that can directly drive the class is chosen to drive the cache. If multiple adapters return
63 * SupportType::Direct then the first is chosen. If multiple adapters return SupportType::Derived then the
64 * adapter with the closest desired class will be chosen (closest in terms of inheritance hierarchy between the
65 * desired direct class and the provided class)
66 *
67 * If the adapter can only handle one specific type then only using SupportType::Direct and SupportType::None
68 * will make sure it only receives calls from the cache if exactly that component class is selected
69 *
70 * @param InComponent - the component class to test
71 *
72 * @return the type of support this adapter can give for the specified class
73 */
74 virtual SupportType SupportsComponentClass(UClass* InComponentClass) const = 0;
75
76 /**
77 * Query the class that this adapter wants to drive. This will be used to work out the most likely candidate for
78 * an adapter when multiple adapters return SupportType::Derived. For example in the following hierarchy:
79 *
80 * UPrimitiveComponent
81 * |
82 * UDerivedComponentA
83 * |
84 * UDerivedComponentB
85 *
86 * An adapter directly supporing UDerivedComponentA that returns SupportType::Derived for a class of
87 * UDerivedComponentB will be picked over an adapter that directly supports UPrimitiveComponent and returns
88 * SupportType::Derived for a class of UDerivedComponentB
89 *
90 * Note: All classes returned from this MUST have UPrimitiveComponent in their hierarchy as that is the base
91 * physically-capable component
92 */
93 virtual UClass* GetDesiredClass() const = 0;
94
95 /**
96 * Gets the priority for an adapter.
97 * When two or more adapters give the same support level for a given component class this priority will be
98 * used to decide which adapter will be used. All base engine level adapters per type will use a priority
99 * between EngineAdapterPriorityBegin and UserAdapterPriorityBegin defined in this interface.
100 *
101 * Users implementing adapters intended to override all engine functionality should return priorities
102 * above UserAdapterPriorityBegin to ensure they will always be selected above engine adapters.
103 */
104 virtual uint8 GetPriority() const = 0;

179 * This must return a stable GUID that uniquely identifies any derived classes. This GUID is embedded into
