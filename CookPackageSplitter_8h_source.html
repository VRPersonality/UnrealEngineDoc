79 * An ICookPackageSplitter for a single generator package normally is constructed only once and handles
80 * all generated packages for that generator, but during MPCook in cases of load balancing between CookWorkers,
81 * it is possible that the original splitter is destructed but then recreated later. This is guaranteed not
82 * to happen without a GarbageCollection pass in between, but that GarbageCollection may fail to destruct the
83 * generator package if it is still referenced from other packages or systems. Depending on the ICookPackageSplitter's
84 * implemenation, this failure to GC might cause an error, because changes made from the previous splitter are not

93 /**
94 * Return value for the DoesGeneratedRequireGenerator function. All levels behave correctly, but provide
95 * different tradeoffs of guarantees to the splitter versus performance.
96 */
97 enum class EGeneratedRequiresGenerator : uint8
98 {
99 /**
100 * GetGenerateList will be called before PopulateGeneratedPackage. PopulateGenerator and PreSaveGenerator
101 * might or might not be called before. OutKeepReferencedPackages from PopulateGenerator will not be kept
102 * referenced after PostSaveGenerator. Best for performance.
103 */
104 None,
105 /**
106 * GetGenerateList and PopulateGenerator will be called before PopulateGeneratedPackage.
107 * OutKeepReferencedPackages from PopulateGenerator will be kept referenced until all generated and generator
108 * packages call PostSave or until the splitter is destroyed. Performance cost: Possible extra calls to
109 * PopulateGeneratedPackage, possible unnecessary memory increase due to OutKeepReferencedPackages.
110 */
111 Populate,
112 /**
113 * GetGenerateList PopulateGenerator, PreSaveGenerator, and PostSaveGenerator will be called before
114 * PopulateGeneratedPackage. Performance cost: Progress on generated packages will be delayed until generator
115 * finishes saving. Possible unnecessary memory increase due to OutKeepReferencedPackages. Retraction is not
116 * possible in MPCook for the generated packages; they must all be saved on the same CookWorker that saves the
117 * generator.
118 */
119 Save,
120 Count,
121 };
122 /**
123 * Return capability setting which indicates which splitter functions acting on the parent generator package must
124 * be called on the splitter before splitter functions acting on the generated packages can be called. Also impacts
125 * the lifetime of memory guarantees for the generator functions. @see EGeneratedRequiresGenerator. Default is
126 * EGeneratedRequiresGenerator::None, which provides the best performance but the fewest guarantees.
127 *
128 * Examples of dependencies and what capability level should be used:
129 * ShouldSplit call reads data that is written by BeginCacheForCookedPlatformData:
130 * EGeneratedRequiresGenerator::Save
131 * PopulateGeneratedPackage or PreSaveGeneratedPackage read data that is written by PopulateGeneratorPackage:
132 * EGeneratedRequiresGenerator::Populate
133 */
134 virtual EGeneratedRequiresGenerator DoesGeneratedRequireGenerator()
135 {
136 return EGeneratedRequiresGenerator::None;
137 }
