28/**
29 * A mechanic for displaying a sequence of control points and moving them about. Has an interactive initialization mode for
30 * first setting the points.
31 *
32 * When editing, hold shift to select multiple points. Hold Ctrl to add an extra point along an edge. To add points to either end of
33 * the sequence, first select either the first or last point and then hold Ctrl.
34 * Backspace deletes currently selected points. In edit mode, holding Shift generally toggles the snapping behavior (makes it opposite
35 * of the current SnappingEnabled setting), though this is not yet implemented while the gizmo is being dragged.
36 *
37 * TODO:
38 * - Make it possible to open/close loop in edit mode
39 * - Improve display of occluded control points (checkerboard the material)
40 * - Allow deselection of vertices by clicking away?
41 * - Lump the point/line set components into PreviewGeometryActor.
42 */
43UCLASS(MinimalAPI)
44class UCurveControlPointsMechanic : public UInteractionMechanic, public IClickBehaviorTarget, public IHoverBehaviorTarget

50 // We want some way to store the control point sequence that lets us easily associate points with their renderable and hit-testable
51 // representations, since we need to alter all of these together as points get moved or added. We use FOrderedPoints for this, until
52 // we decide on how we want to store sequences of points in general.
53 // FOrderedPoints maintains a sequence of point ID's, the positions associated with each ID, and a mapping back from ID to position.
54 // The ID's can then be used to match to renderable points, hit-testable points, and segments going to the next point.
55
