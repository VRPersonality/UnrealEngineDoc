22/**
23 * The actual asset editor class doesn't have that much in it, intentionally.
24 *
25 * Our current asset editor guidelines ask us to place as little business logic as possible
26 * into the class, instead putting as much of the non-UI code into the subsystem as possible,
27 * and the UI code into the toolkit (which this class owns).
28 *
29 * However, since we're using a mode and the Interactive Tools Framework, a lot of our business logic
30 * ends up inside the mode and the tools, not the subsystem. The front-facing code is mostly in
31 * the asset editor toolkit, though the mode toolkit has most of the things that deal with the toolbar
32 * on the left.
33 */
