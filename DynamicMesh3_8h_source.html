58/**
59* FDynamicMesh3 is a dynamic triangle mesh class. The mesh has has connectivity,
60* is an indexed mesh, and allows for gaps in the index space.
61*
62* internally, all data is stored in POD-type buffers, except for the vertex->edge
63* links, which are stored as List<int>'s. The arrays of POD data are stored in
64* TDynamicVector's, so they grow in chunks, which is relatively efficient. The actual
65* blocks are arrays, so they can be efficiently mem-copied into larger buffers
66* if necessary.
67*
68* Reference counts for verts/tris/edges are stored as separate FRefCountVector
69* instances.
70*
71* Vertices are stored as doubles, although this should be easily changed
72* if necessary, as the internal data structure is not exposed
73*
74* Per-vertex Vertex Normals, Colors, and UVs are optional and stored as floats.
75* Note that in practice, these are generally only used as scratch space, in limited
76* circumstances, usually when needed for performance reasons. Most of our geometry
77* code instead prefers to read attributes from the per-triangle AttributeSet accessed
78* via Attributes() (see TDynamicMeshOverlay for a description of the structure). For
79* instance, an empty (but existing) attribute set will take precedence over non-empty
80* vertex normals in much of our processing code.
81*
82* For each vertex, VertexEdgeLists[i] is the unordered list of connected edges. The
83* elements of the list are indices into the edges list.
84* This list is unsorted but can be traversed in-order (ie cw/ccw) at some additional cost.
85*
86* Triangles are stored as 3 ints, with optionally a per-triangle integer group id.
87* The group IDs stored here DO get widely used and preserved in our geometry code
88* (unlike the per-vertex attributes described earlier), even though the AttributeSet
89* can store group IDs as well (potentially in multiple layers).
90*
91* The edges of a triangle are similarly stored as 3 ints, in triangle_edes. If the
92* triangle is [v1,v2,v3], then the triangle edges [e1,e2,e3] are
93* e1=edge(v1,v2), e2=edge(v2,v3), e3=edge(v3,v1), where the e# are indexes into edges.
94*
95* Edges are stored as tuples of 4 ints. If the edge is between v1 and v2, with neighbour
96* tris t1 and t2, then the edge is [min(v1,v2), max(v1,v2), t1, t2]. For a boundary
97* edge, t2 is InvalidID. t1 is never InvalidID.
98*
99* Most of the class assumes that the mesh is manifold. Many functions will
100* work if the topology is non-manifold, but behavior of operators like Split/Flip/Collapse
101* edge is untested.
102*
103* The function CheckValidity() does extensive sanity checking on the mesh data structure.
104* Use this to test your code, both for mesh construction and editing!!
105*/
