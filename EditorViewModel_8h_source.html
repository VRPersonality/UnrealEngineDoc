28/**
29 * This represents to root view-model for a sequencer-like editor.
30 *
31 * This view-model assumes that the editor includes at least an outliner area and a track area.
32 * Other panels in the editor can be added to the list of panels.
33 *
34 * The data being edited is represented by a root view-model which doesn't change: if the editor
35 * needs to edit a different piece of data, this should change *inside* the root view-model, and
36 * call the necessary code to refresh the entire view-model hierarchy.
37 * Note that this root view-model is *not* parented under the editor view-model.
38 */
