183/**
184* Generalized source of Root Motion to a CharacterMovementComponent.
185*
186* Has priorities and different accumulation modes (override, additive, set, ignore).
187*
188* RootMotionSources are accumulated from highest priority to lowest, once all values
189* are "finalized" (meaning no more accumulation to those values), root motion accumulation
190* is finished. Having all of translation and rotation in a transform being overridden during
191* root motion accumulation means that CharacterMovementComponent Phys*() functions no longer
192* need to CalcVelocity() since it'll just get overridden anyway.
193*
194* Some example source possibilities:
195* Full Animation Root Motion (matching only form of root motion we've supported historically):
196* - Overrides all translation axes or rotation or both
197* - Very High priority so it gets applied first and since it is override no other
198* RootMotionSources have an influence that are lower priority
199* - Scale on translation and/or rotation is set to 1 to match animation 1 for 1
200* - Provides local space transform (character-orientation relative)
201*
202* - End result: Same as before - overrides velocity/rotation to values in AnimMontage
203* - But can now scale individual translation axes if desired, or instead of override
204* can use "set" to allow other additive sources of root motion to be applied on top
205* of the animation root motion (modify velocity to home root motion melee attacks, etc.)
206*
207* Jump pad volumes that work during animation root montage attacks
208* - Constant velocity modification Z up, additive and high priority
209* - Attack/melee animation root motion uses "set" instead of "override" to allow other sources
210* - Animation root motion is applied, then the Z up additive of this source is so you are
211* affected by the jump pad
212*
213* Attack/melee homing
214* - Additive, lower priority so it gets applied after any animation/significant root motion
215* - World space, sourced by logic modifying velocity to go towards nearby enemies
216* - Could point to a UAbilityTask node from the GameplayAbilities system
217*
218* Character movement controlled by splines/curves in-editor instead of requiring animation tweaks
219* - Same as animation root motion but source from spline/curve data
220*
221*/
