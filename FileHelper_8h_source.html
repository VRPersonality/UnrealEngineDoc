81 /**
82 * Load a binary file a block at a time into a buffer and call the visitor each time the buffer is populated
83 * with a new block. This uses less memory and has higher cache coherency than loading the entire file into
84 * a block of bytes, and is more performant than reading the file a byte at a time. Useful for e.g. hashing or
85 * parsing.
86 *
87 * @param Filename The file to read
88 * @param BlockVisitor Function called repeatedly with each sequential block.
89 * Size of the input might change for performance and should not be assumed. Possibly the entire file will
90 * be passed in a single block. If LoadFileInBlocks returns true, the sum of sizes in each call to
91 * BlockVisitor equals the size of the requested range of the the file on disk. If the requested range
92 * has size 0, LoadFileInBlocks will return true and the BlockVisitor will not be called.
93 * @param Offset Start of the range of bytes to read from the file. If less than zero or larger than filesize,
94 * it will be silently clamped to the [0,FileSize] range. This may result in LoadFileInBlocks returning true
95 * but using an empty range and not calling BlockVisitor.
96 * @param Length of the range of bytes to read from the file. If less than zero, all bytes between Offset and end
97 * of file are read. If larger than the size of the range between Offset and end of file, it will be
98 * silently shortened to the size of the range.
99 * @param Flags Flags to pass to IFileManager::CreateFileReader
100 * @param BlockSize Size of the internally-allocated buffer to read bytes into and pass to each call to
101 * BlockVisitor. Sizes <= 0 use the default-selected block size for the current platform.
102 */
103 static CORE_API bool LoadFileInBlocks(FStringView Filename, TFunctionRef<void(FMemoryView)> BlockVisitor,
104 int64 Offset = 0, int64 Size = -1, uint32 Flags = 0, int64 BlockSize = 0);

187 * Saves a 24/32Bit BMP file to disk for debug image dump purposes
188 *
189 * for general image saving (to BMP or any other format); use FImageUtils::SaveImage instead
190 * CreateBitmap is mainly for debug dump images

192 * note this also calls SendDataToPCViaUnrealConsole
193 * and uses GenerateNextBitmapFilename if Pattern does not have ".bmp" on it
194 *
195 * @param Pattern filename with path, must not be 0, if with "bmp" extension (e.g. "out.bmp") the filename stays like this, if without (e.g. "out") automatic index numbers are addended (e.g. "out00002.bmp")
196 * @param DataWidth - Width of the bitmap supplied in Data >0
197 * @param DataHeight - Height of the bitmap supplied in Data >0
198 * @param Data must not be 0
199 * @param SubRectangle optional, specifies a sub-rectangle of the source image to save out. If NULL, the whole bitmap is saved
200 * @param FileManager must not be 0
201 * @param OutFilename optional, if specified filename will be output
202 * @param bInWriteAlpha optional, specifies whether to write out the alpha channel. Will force BMP V4 format.
203 * @param ColorChannel optional, specifies a specific channel to write out (will be written out to all channels gray scale).
204 *
205 * @return true if success
206 */
207 static CORE_API bool CreateBitmap( const TCHAR* Pattern, int32 DataWidth, int32 DataHeight, const struct FColor* Data, FIntRect* SubRectangle = NULL, IFileManager* FileManager = &IFileManager::Get(), FString* OutFilename = NULL, bool bInWriteAlpha = false, EColorChannel ColorChannel = EColorChannel::All);
