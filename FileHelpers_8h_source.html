468 /**
469 * Optionally prompts the user for which of the provided packages should be saved, and then additionally prompts the user to check-out any of
470 * the provided packages which are under source control. If the user cancels their way out of either dialog, no packages are saved. It is possible the user
471 * will be prompted again, if the saving process fails for any reason. In that case, the user will be prompted on a package-by-package basis, allowing them
472 * to retry saving, skip trying to save the current package, or to again cancel out of the entire dialog. If the user skips saving a package that failed to save,
473 * the package will be added to the optional OutFailedPackages array, and execution will continue. After all packages are saved (or not), the user is provided with
474 * a warning about any packages that were writable on disk but not in source control, as well as a warning about which packages failed to save.
475 *
476 * @param PackagesToSave The list of packages to save. Both map and content packages are supported
477 * @param bCheckDirty If true, only packages that are dirty in PackagesToSave will be saved
478 * @param bPromptToSave If true the user will be prompted with a list of packages to save, otherwise all passed in packages are saved
479 * @param Title If bPromptToSave true provides a dialog title
480 * @param Message If bPromptToSave true provides a dialog message
481 * @param OutFailedPackages [out] If specified, will be filled in with all of the packages that failed to save successfully
482 * @param bAlreadyCheckedOut If true, the user will not be prompted with the source control dialog
483 * @param bCanBeDeclined If true, offer a "Don't Save" option in addition to "Cancel", which will not result in a cancellation return code.
484 *
485 * @return An enum value signifying success, failure, user declined, or cancellation. If any packages at all failed to save during execution, the return code will be
486 * failure, even if other packages successfully saved. If the user cancels at any point during any prompt, the return code will be cancellation, even though it
487 * is possible some packages have been successfully saved (if the cancel comes on a later package that can't be saved for some reason). If the user opts the "Don't
488 * Save" option on the dialog, the return code will indicate the user has declined out of the prompt. This way calling code can distinguish between a decline and a cancel
489 * and then proceed as planned, or abort its operation accordingly.
490 */
