210 /**
211 * During time discrepancy resolution, we "pay back" the time discrepancy at this rate for future moves until total error is zero.
212 * 1.0 = 100% resolution rate, meaning the next X ServerMoves from the client are fully paying back the time,
213 * 0.5 = 50% resolution rate, meaning future ServerMoves will spend 50% of tick continuing to move the character and 50% paying back.
214 * Lowering from 100% could be used to produce less severe/noticeable corrections, although typically we would want to correct
215 * the client as quickly as possible.
216 */
217 UPROPERTY(GlobalConfig)
218 float MovementTimeDiscrepancyResolutionRate;
219
220 /**
221 * Accepted drift in clocks between client and server as a percent per second allowed.
222 *
223 * 0.0 is "no forgiveness" and all logic would run on raw values, no tampering on the server side.
224 * 0.02 would be a 2% per second difference "forgiven" - if the time discrepancy in a given second was less than 2%,
225 * the error handling/detection code effectively ignores it.
226 *
227 * Increasing this value above 0% lessens the chance of false positives on time discrepancy (burst packet loss, performance
228 * hitches), but also means anyone tampering with their client time below that percent will not be detected and no resolution
229 * action will be taken, and anyone above that threshold will still gain the advantage of this % of time boost (if running at
230 * 10% speed-up and this value is 0.05 or 5% allowance, they would only be resolved down to a 5% speed boost).
231 *
232 * Time discrepancy detection code DOES keep track of LifetimeRawTimeDiscrepancy, which is unaffected by this drift allowance,
233 * so cheating below DriftAllowance percent could be tracked and acted on outside of an individual game. For example, if DriftAllowance
234 * was 0.05 (meaning we're not going to actively prevent any cheating below 5% boosts to ensure less false positives for normal players),
235 * we could still post-process analytics of the game showing that Player X regularly runs at 4% speed boost and take action.
236 */
237 UPROPERTY(GlobalConfig)
238 float MovementTimeDiscrepancyDriftAllowance;
