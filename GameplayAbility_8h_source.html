40/**
41 * UGameplayAbility
42 *
43 * Abilities define custom gameplay logic that can be activated or triggered.
44 *
45 * The main features provided by the AbilitySystem for GameplayAbilities are:
46 * -CanUse functionality:
47 * -Cooldowns
48 * -Costs (mana, stamina, etc)
49 * -etc
50 *
51 * -Replication support
52 * -Client/Server communication for ability activation
53 * -Client prediction for ability activation
54 *
55 * -Instancing support
56 * -Abilities can be non-instanced (native only)
57 * -Instanced per owner
58 * -Instanced per execution (default)
59 *
60 * -Basic, extendable support for:
61 * -Input binding
62 * -'Giving' abilities (that can be used) to actors
63 *
64 *
65 * See GameplayAbility_Montage for an example of a non-instanced ability
66 * -Plays a montage and applies a GameplayEffect to its target while the montage is playing.
67 * -When finished, removes GameplayEffect.
68 *
69 * Note on replication support:
70 * -Non instanced abilities have limited replication support.
71 * -Cannot have state (obviously) so no replicated properties
72 * -RPCs on the ability class are not possible either.
73 *
74 * To support state or event replication, an ability must be instanced. This can be done with the InstancingPolicy property.
75 */

121 // ----------------------------------------------------------------------------------------------------------------
122 //
123 // The important functions:
124 //
125 // CanActivateAbility() - const function to see if ability is activatable. Callable by UI etc
126 //
127 // TryActivateAbility() - Attempts to activate the ability. Calls CanActivateAbility(). Input events can call this directly.
128 // - Also handles instancing-per-execution logic and replication/prediction calls.
129 //
130 // CallActivateAbility() - Protected, non virtual function. Does some boilerplate 'pre activate' stuff, then calls ActivateAbility()
131 //
132 // ActivateAbility() - What the abilities *does*. This is what child classes want to override.
133 //
134 // CommitAbility() - Commits reources/cooldowns etc. ActivateAbility() must call this!
135 //
136 // CancelAbility() - Interrupts the ability (from an outside source).
137 //
138 // EndAbility() - The ability has ended. This is intended to be called by the ability to end itself.
139 //
140 // ----------------------------------------------------------------------------------------------------------------
