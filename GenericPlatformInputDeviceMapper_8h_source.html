15/**
16 * Base class to private a mapping of Platform Users (FPlatformUserID)
17 * to their associated available input devices (FInputDeviceID).
18 *
19 * This will handle the allocation of the globally unique identifier
20 * of the FInputDeviceID, and allow overrides of how each platform
21 * maps input devices to their users. Some platforms may desire to
22 * have each new input device assigned to a different user, while others
23 * may want multiple input devices associated with a single user.
24 *
25 * @note Keep any function comments up to date with those in PlatformInputDeviceMapperLibrary.h!
26 */

227 /**
228 * Remap the legacy "int32 ControllerId" to the updated FPlatformUserId and FInputDeviceId. Use this function to
229 * add compatibility to platforms that may not have implemented this device mapper yet.
230 *
231 * This is useful for functions such as FGenericApplicationMessageHandler::OnControllerAnalog that used to
232 * use the "int32 ControllerId" as a parameter so that you can call the new FGenericApplicationMessageHandler
233 * that take in a PlatformUserId and an InputDeviceId.
234 *
235 * @param ControllerId The old plain "int32" that represented gamepad id or user index depending on the context.
236 * @param InOutUserId If the old function provides a PlatformId then pass it here, otherwise pass PLATFORMUSERID_NONE.
237 * @param OutInputDeviceId The best guess for an InputDeviceId based on the legacy int32 ControllerId. This may be INPUTDEVICEID_NONE.
238 * @return True if this maps to a real user
239 */
