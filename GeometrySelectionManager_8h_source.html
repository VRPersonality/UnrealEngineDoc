48/**
49 * UGeometrySelectionManager provides the infrastructure for "Element Selection", ie
50 * geometric sub-elements of some geometry object like a Triangle Mesh. The Manager is
51 * designed to work with a relatively vague concept of "element", so it doesn't explicitly
52 * reference triangles/etc, and the selectable-elements and how-elements-are-selected
53 * concepts are provided by abstract-interfaces that allow various implememtations.
54 *
55 * The "Geometry Objects", eg like a DynamicMeshComponent, Gameplay Volume, etc, are
56 * referred to as "Active Targets" in the Manager. External code provides and updates
57 * the set of Active Targets, eg for example tracking the active Actor Selection in the Editor.
58 *
59 * For a given Target, a tuple (Selector, Selection, SelectionEditor) is created and maintained.
60 * The FGeometrySelection is ultimately a basic list of integers and does not have any knowledge
61 * of what it is a selection *of*, and is not intended to be directly edited. Instead the
62 * SelectionEditor provides that functionality. This separation allows "selection editing" to
63 * be customized, eg to enforce invariants or constraints that might apply to certain kinds of selections.
64 *
65 * The IGeometrySelector provides the core implementation of what "selection" means for a given
66 * Target, eg like a mesh Component, or mesh object like a UDynamicMesh. The Selector is
67 * created by a registered Factory, allowing client code to provide custom implementations for
68 * different Target Types. Updates to the Selection are done via the Selector, as well as queries
69 * about (eg) renderable selection geometry. 3D Transforms are also applied via the Selector,
70 * as only it has the knowledge about what can be transformed and how it can be applied.
71 *
72 * The GeometrySelectionManager provides high-level interfaces for this system, for example
73 * external code (eg such as something that creates a Gizmo for the active selection) only
74 * needs to interact with SelectionManager, calling functions like
75 * ::BeginTransformation() / ::UpdateTransformation() / ::EndTransformation().
76 * The SelectionManager also handles Transactions/FChanges for the active Targets and Selections.
77 *
78 */
