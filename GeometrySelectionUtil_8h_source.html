340/**
341 * Convert Selection from one type to another, based on geometry/topology types in FromSelectionIn and ToSelectionOut.
342 *
343 * The following table describes the conversions, the FromSelectionIn/ToSelectionOut type are rows/columns respectively:
344 *
345 * ================================================================
346 * To: Triangle Polygroup
347 * From: Vertex Edge Face Vertex Edge Face
348 * ----------------------------------------------------------------
349 * Triangle Vertex 1 1 7 4# 7# 7#
350 * Triangle Edge 1 1 8 4# 1# 8#
351 * Triangle Face 1 1 1 4# 3# 2#
352 * Polygroup Vertex 6# 9# 9 1 7# 7#
353 * Polygroup Edge 5# 1# 8 1# 1 8#
354 * Polygroup Face 1# 1# 1 1# 1# 1
355 * ================================================================
356 *
357 * Key:
358 * . These conversions are not implemented... yet? GroupTopology is ignored
359 * 1 - The implementation is obvious/unambiguous
360 * 2 - Polygroup faces containing any input triangle are selected
361 * 3 - Polygroup edges containing any input triangle edge are selected, but
362 * polygroup edges containing only input triangle vertices are not.
363 * 4 - Polygroup corners coinciding with any input triangle face/edge/vertex are selected
364 * 5 - All mesh vertices along the polygroup edge are selected
365 * 6 - All mesh vertices coinciding with polygroup corners are selected
366 * 7 - All Elements which immediately touch any input vertex/corner are selected
367 * 8 - All triangles or faces on either side of the any input edge are selected
368 * PolyEdge->TriFace includes all triangles in both groups which the edge borders
369 * 9 - All edges in each PolyEdge with touch any input Corner are selected
370 * All triangles in each PolyFace which touch any input Corner are selected
371 * # indicates GroupTopology must not be null for this combination. If this symbol is missing GroupTopology is ignored
372 *
373 * @return true if conversion is supported and was computed successfully, return false otherwise
374 */

497/**
498 * Given a selection, return the vertex IDs of the vertices on the boundary of this selection. A selected vertex
499 * is considered to be on the boundary either if it is on the actual mesh boundary (for an open mesh) or it is
500 * connected to a triangle element that is not part of the selection (i.e. if the vertex has a neighbor vertex
501 * not in selection for a selection of type EGeometryElementType::Vertex, or an adjacent edge not in selection
502 * for a selection of type EGeometryElementType::Edge, an adjacent triangle not in the selection for a selection
503 * of type EGeometryElementType::Face).
504 *
505 * For selections of type EGeometryTopologyType::Polygroup, the results are equivalent to first converting the
506 * selection to corresponding EGeometryTopologyType::Triangle selection and then finding the boundary vertices.
507 * This gives the intuitive result for face selections, but may or may not be what is desired for polygroup
508 * vertex/edge selections, because vertices/edges that seem to be on the interior of the polygroup selection
509 * may be considered border vertices if the tesselation is such that they are adjacent to unselected triangle
510 * vertices/edges.
511 *
512 * @param GroupTopology Must not be null for selections of type EGeometryTopologyType::Polygroup
513 * @param BorderVidsOut Output vertex IDs of border vertices
514 * @param CurVerticesOut Output vertex IDs of all vertices in the current selection
515 * @return true if successful. For instance, could fail if GroupTopology was null for a EGeometryTopologyType::Polygroup selection
516 */

521 TSet<int32>& BorderVidsOut, TSet<int32>& CurVerticesOut);
