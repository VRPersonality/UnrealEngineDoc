147/**
148 * IGeometrySelectionTransformer is a transient object that is created by an IGeometrySelector
149 * to provide external code with a way to manipulate/transform an active Selection.
150 * This allows (eg) a Gizmo to be hooked up to an active Selection without either
151 * explicitly knowing about each other.

154 * IGeometrySelector::InitializeTransformation() will create a suitable IGeometrySelectionTransformer

201/**
202 * IGeometrySelector is a base API definition for, roughly, "an object that knows how to
203 * select elements and/or work with element selections on some other object", where "element"
204 * is some "part" of the object, like a mesh edge/vertex/triangle, or intermediate structure (eg a polygroup).
205 *
206 * Essentially the purpose of IGeometrySelector is to provide the bridge between "object that can have a selection"
207 * and "the active selection on that object". The Selection is stored as a FGeometrySelection which has
208 * no reference to a type or target object, and standard target objects (eg a StaticMesh, Volume, etc)
209 * have no inherent knowledge of, or support for, low-level geometry selections.
210 * So the Selector provides these capabilities, as well as related functionality like
211 * being able to transform an active selection and emit necessary transactions/etc.
212 *
213 * Generally clients will not create a IGeometrySelector instance directly. For a given
214 * implmentation, a IGeometrySelectorFactory is registered with the UGeometrySelectionManager,
215 * and it creates instances based on the active selection.
216 *
217 * Where possible, the UGeometrySelectionManager will also "sleep" a IGeometrySelector for a given
218 * target object instead of destroying it when the object is deselected, and then "restore"
219 * it if the object is re-selected. This avoids frequent churn in Selectors as the user
220 * clicks between objects.
221 *
222 * Selector implementations should ideally have a strategy to lazy-construct any large data
223 * structures (eg mesh AABBTrees, etc). Again this is to avoid excessive overhead when the
224 * user (for example) selects a large object but has no intention of doing element-level selection.
225 *
226 */
227class IGeometrySelector
228{
229public:
230 using FGeometrySelection = UE::Geometry::FGeometrySelection;
231 using FGeometrySelectionEditor = UE::Geometry::FGeometrySelectionEditor;
232 using FGeometrySelectionBounds = UE::Geometry::FGeometrySelectionBounds;
233 using FGeometrySelectionElements = UE::Geometry::FGeometrySelectionElements;
234 using FGeometrySelectionUpdateConfig = UE::Geometry::FGeometrySelectionUpdateConfig;
235 using FGeometrySelectionUpdateResult = UE::Geometry::FGeometrySelectionUpdateResult;

255 * @return true if Sleep was successful, if not the Selector will likely be Shutdown

257 virtual bool Sleep() { return false; }
