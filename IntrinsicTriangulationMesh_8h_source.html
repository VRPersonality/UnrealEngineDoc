20/**
21* Intrinsic Meshes:
22*
23* An Intrinsic Mesh can be thought of as a mesh that overlays another mesh, sharing
24* the original surface. The edges of the intrinsic mesh are constrained to be on the original mesh surface,
25* but need not align with the original mesh edges. The lengths of the intrinsic edges are measured on
26* the surface of the original mesh (not the R3 distance). The original mesh vertices are a subset of
27* the intrinsic mesh vertices and intrinsic edge splits and triangle pokes may introduce new intrinsic mesh vertices.
28*
29* Note: the implementation is a simple triangle-based mesh but there is no restriction that
30* edge(a, b) is unique (e.g. multiple intrinsic edges may connect the same two vertices with different paths).
31* In fact this mesh allows triangles and edges with repeated vertices. Such structures arise naturally
32* with intrinsic triangulation, for example an edge that starts and ends at the
33* same vertex may encircle a mesh.
34*
35* The bookkeeping that manages the correspondence between locations on the intrinsic mesh and on the surface mesh
36* is implemented either using integer-based "Normal Coordinates"
37* ( cf 'Integer Coordinates for Intrinsic Geometry Processing' M. Gillespie, N. Sharp, and K. Crane, TOG , Vol. 40, December 2021. )
38* as used by FIntrinsicEdgeFlipMesh and FIntrinsicMesh;
39* or alternately floating-point based generalized directions "Signpost data"
40* ( cf 'Signpost Coordinates inspired by "Navigating Intrinsic Triangulations' Sharp, Soliman and Crane [2019, ACM Transactions on Graphics])
41* as used by FIntrinsicTriangulation.
42*
43*/
44
45/**
46* The FSimpleIntrinsicEdgeFlipMesh is designed to work with the function FlipToDelaunay() to make an
47* Intrinsic Delaunay Triangulation (IDT) of the same surface as a given FDynamicMesh,
48* allowing for more robust cotangent-Laplacians (see LaplacianMatrixAssembly.h)
49*
50*
51* The vertices in this FSimpleIntrinsicEdgeFlipMesh are exactly those found in the surface mesh
52* since it supports edge flips, but no other operations that change the mesh connectivity,
53* and no operations that affect the vertices.
54*
55* This mesh alone does not support computing the intersections of intrinsic edges with the surface mesh edges
56* and as a result it can not be used to easily extract the path of an intrinsic edge.
57* To do such computations FIntrinsicEdgeFlipMesh, FIntrinsicMesh or FIntrinsicTriangulation should be used.
58*
59* The API of FIntrinsicEdgeFlipMesh is similar to FDynamicMesh3, but only has a minimal subset of methods and some
60* such as ::FlipEdge() and ::GetEdgeOpposingV() have very different implementations.
61*
62* In addition to managing vertices, triangles and edges this class also tracks edge lengths (as measured on the surface mesh)
63* and for convenience the internal angles for each triangle.
64*/

605/**
606* FIntrinsicEdgeFlipMesh augments the base class FSimpleIntrinsicEdgeFlipMesh with
607* the addition of a 'normal coordinate' based correspondence with the surface mesh.
608*
609* This class supports both the trace of intrinsic mesh edges across the surface mesh,
610* and the trace of surface mesh edges across the intrinsic mesh.
611*
612* Note, because only edge flips are supported (no edge split or triangle pokes), the
613* vertex set of the intrinsic mesh and the surface (i.e. extrinsic) mesh will be the same.
614*
615* The expected use
616*
617* // generate the implicit mesh from a surface mesh
618* FIntrinsicEdgeFlipMesh ImplicitMesh( DynamicMesh );
619*
620* .. do some edge flips .. perhaps FlipToDelaunay( ImplicitMesh, ..)
