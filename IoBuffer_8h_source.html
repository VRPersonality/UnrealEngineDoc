56 /**
57 * Relinquishes control of the internal buffer to the caller and removes it from the FIoBuffer.
58 * This allows the caller to assume ownership of the internal data and prevent it from being deleted along with
59 * the FIoBuffer.
60 *
61 * NOTE: It is only valid to call this if the FIoBuffer currently owns the internal memory allocation, as the
62 * point of the call is to take ownership of it. If the FIoBuffer is only wrapping the allocation then it will
63 * return a failed FIoStatus instead.
64 *
65 * @return A status wrapper around the memory pointer. Even if the status is valid the pointer might still be null.
66 */
67 [[nodiscard]] CORE_API TIoStatusOr<uint8*> Release();
