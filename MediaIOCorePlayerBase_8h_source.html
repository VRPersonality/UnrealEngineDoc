61/**
62 * Implements a base player for hardware IO cards.
63 *
64 * The processing of metadata and video frames is delayed until the fetch stage
65 * (TickFetch) in order to increase the window of opportunity for receiving
66 * frames for the current render frame time code.
67 *
68 * Depending on whether the media source enables time code synchronization,
69 * the player's current play time (CurrentTime) is derived either from the
70 * time codes embedded in frames or from the Engine's global time code.
71 */
