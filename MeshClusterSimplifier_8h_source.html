75/**
76 * Makes a simplified copy of the input mesh
77 *
78 * This cluster simplify method first clusters vertices locally by distance (calculated along mesh edges), and creates new triangles
79 * from the connectivity of the clusters. i.e., it is a triangulation of the dual of the graph voronoi diagram over mesh edges.
80 *
81 * To preserve feature edges:
82 * (1) constrained / feature-edge vertices are prioritized as cluster 'seeds,' and
83 * (2) clusters are grown along feature edges first, then free edges after -- and growth over 'free' edges cannot claim 'constrained' vertices.
84 * This locks in clusters along 'constrained' feature edges.
85 *
86 * Note that mesh features can be lost if the clusters are large enough that the graph becomes degenerate
87 * -- e.g., if a mesh island has so few clusters that the graph connectivity does not contain triangles.
88 *
89 *
90 * @param InMesh The mesh to simplify
91 * @param OutSimplifiedMesh This mesh will store the simplified result mesh
92 * @param SimplifyOptions Options controlling simplification
93 * @return true on success
94 */
95bool DYNAMICMESH_API Simplify(const FDynamicMesh3& InMesh, FDynamicMesh3& OutSimplifiedMesh, const FSimplifyOptions& SimplifyOptions);
