93/**
94 * UMeshElementsVisualizer is a subclass of UPreviewGeometry that displays mesh elements.
95 * Currently supports wireframe, boundary edges, UV seams, Normal seams and Color seams.
96 *
97 * UMeshElementsVisualizer initializes an instance of UMeshElementsVisualizerProperties
98 * as its .Settings value, and will watch for changes in these properties.
99 *
100 * Mesh is accessed via lambda callback provided by creator/client. See SetMeshAccessFunction() comments
101 */

109 /**
110 * A reference to a function that can be called with a const FDynamicMesh3 to do some computation on it.
111 */
112 typedef TFunctionRef<void(const UE::Geometry::FDynamicMesh3& Mesh)> ProcessDynamicMeshFunc;
113
114 /**
115 * UMeshElementsVisualizer needs access to a mesh to generate it's data structures. However
116 * we may not be able to (or shouldn't) directly pass a pointer to UMeshElementsVisualizer.
117 * For example a UDynamicMeshComponent can expose it's internal mesh via a function ProcessMesh( TFunctionRef<void(const FDynamicMesh3&)> Func ).
118 * UMeshElementsVisualizer cannot directly call that function without knowing about the UDynamicMeshComponent,
119 * which we would like to avoid. So instead the client of UMeshElementsVisualizer provides a
120 * MeshAccessFunction that we can call with an internal function/lambda that UMeshElementsVisualizer
121 * internals will produce, that can be passed a const FDynamicMesh3& for processing (this is a ProcessDynamicMeshFunc).
122 * The job of the MeshAccessFunction is to call this ProcessDynamicMeshFunc on a FDynamicMesh3,
123 * which it may have direct access to, or it may get by calling (eg) UDynamicMeshComponent::ProcessMesh() internally.
124 *
125 * So for example usage with a Tool that has a DynamicMeshComponent could be as follows:
126 * MeshElementsVisualizer->SetMeshAccessFunction([this](UMeshElementsVisualizer::ProcessDynamicMeshFunc ProcessFunc) {
127 * DynamicMeshComponent->ProcessMesh(ProcessFunc);
128 * });
129 * And for usage with a mesh that a Tool class owns:
130 * MeshElementsVisualizer->SetMeshAccessFunction([this](UMeshElementsVisualizer::ProcessDynamicMeshFunc ProcessFunc) {
131 * ProcessFunc(MyInternalMesh);
132 * });
133 */
134 UE_API void SetMeshAccessFunction(TUniqueFunction<void(ProcessDynamicMeshFunc)>&& MeshAccessFunction);
