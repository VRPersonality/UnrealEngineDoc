489 /** MetaSound Binding
490 *
491 * MetaSound IOperators read and write to shared data. For example: a TDataWriteReference<float>
492 * written to by one operator may be read by several other operators holding
493 * onto TDataReadReference<float>. Binding empowers sharing of the underlying
494 * objects (a `float` in the example) between IOperators and minimizing copying
495 * of data. In general, it is assumed that any individual node does not need
496 * to worry about the specific implications of a binding, but can rather assume
497 * that data references are managed by external systems.
498 *
499 * There are 2 known scenarios where an IOperator must be aware of what binding
500 * can do.
501 *
502 * 1. If the IOperator internally caches raw pointers to the underlying data
503 * of a data reference.
504 *
505 * 2. If the IOperator manages multiple internal IOperators with their own
506 * shared connections (e.g. FGraphOperator)
507 *
508 * In the case that a IOperator does need to know manage bound connections, it's
509 * important that the IOperator follows the binding rules.
510 *
511 * General Binding Rules:
512 * - Binding cannot change the access type of an existing data reference.
513 * - IOperators will ignore new TDataValueReferences because Value references
514 * cannot be updated after the operator has been constructed.
515 *
516 * Input Binding Rules:
517 * - Binding an input may update the underlying data object to point to
518 * a new object.
519 * - When updating input pointers of underlying data, a data reference from
520 * and outer scope should replace that of an inner scope. For example, a graph
521 * representing multiple IOperators can override the input pointers of individual
522 * IOperators, but an individual IOperator cannot override the input pointers of
523 * the containing graph.
524 *
525 *
526 * Output Binding Rules:
527 * - Binding an output may NOT update the location of the underlying data object.
528 * - When determining output pointers of underlying data, a data reference from
529 * the inner scope can override the data reference of an outerscope. For example,
530 * a graph representing multiple IOperators cannot override the output pointers
531 * of individual IOperators, but an individual IOperator can override the output
532 * pointers of the containing graph.
533 *
534 *
535 * These rules apply to any method on the FInputVertexInterfaData or
536 * FOutptuVertexInterfaceData which begins with `Bind`
537 */
