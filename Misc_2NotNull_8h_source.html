12/**
13 * TNotNull is a wrapper template which is used to indicate that a pointer or handle is never intended to be null.
14 *
15 * Design and rationale:
16 *
17 * - No default construction, construction with nullptr or comparison against nullptr. Attempting to do so
18 * will cause a compile error.
19 *
20 * - While intended for pointer types, it should be usable for anything else that is nullable, e.g. TFunction.
21 *
22 * - The pointer value is checked on construction or assignment, and then never after that. The only exception
23 * to this is a TNotNull in a moved-from state, which will not re-check the value. - see below.
24 *
25 * - TNotNull is movable. This allows things like TNotNull<TUniquePtr<T>> to be expressed, but means that a
26 * moved-from TNotNull variable can end up being null. However, users are never required to handle a variable
27 * in this state. If a variable is to be reused after it has been moved from (e.g. a data member of an object
28 * that isn't being destroyed) then code that made it null should assign a new non-null value to it before
29 * returning to user code or using it to construct or assign to another TNotNull. Users of your variable should
30 * never be allowed to see a null value. Compilers and static analyzers are allowed to assume that the pointer
31 is not null and optimize and analyze accordingly.
32 *
33 * - The UE_ENABLE_NOTNULL_WRAPPER can be used to disable all checking and become an alias to the inner type, so
34 * there is no runtime overhead at all.
35 */
