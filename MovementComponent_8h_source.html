59/**
60 * MovementComponent is an abstract component class that defines functionality for moving a PrimitiveComponent (our UpdatedComponent) each tick.
61 * Base functionality includes:
62 * - Restricting movement to a plane or axis.
63 * - Utility functions for special handling of collision results (SlideAlongSurface(), ComputeSlideVector(), TwoWallAdjust()).
64 * - Utility functions for moving when there may be initial penetration (SafeMoveUpdatedComponent(), ResolvePenetration()).
65 * - Automatically registering the component tick and finding a component to move on the owning Actor.
66 * Normally the root component of the owning actor is moved, however another component may be selected (see SetUpdatedComponent()).
67 * During swept (non-teleporting) movement only collision of UpdatedComponent is considered, attached components will teleport to the end location ignoring collision.
68 */
