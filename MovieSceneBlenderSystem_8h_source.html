19/**
20 * Base class for all systems that blend data from multiple entities/components into a single entity
21 *
22 * This system has direct coupling to TCompositePropertySystemManager and forms the basis for built-in
23 * blend modes in Sequencer (Absolute, Relative and Additive). Blend 'channels' are allocated in this
24 * system which define a many to one relationship by corresponding blend input(many)/output(one) components
25 * added to the relevant entities. Blend input and output channel components are uint16.
26 *
27 * The class supports a maximum of 65535 blend channels.
28 *
29 * Additionally, blender systems tag their inputs using the built in Absolute, Relative and Additive tags
30 * for efficient computation of each type of blending (this allows each blend type to be computed without branching).
31 *
32 * A simple example of 3 blended floats is as follows:
33 * Entity Data:
34 * Inputs:
35 * float [float component], uint16 [blend channel input], [Absolute Tag] => [ { 100.f, 0 }, { 200.f, 1 } ]
36 * float [float component], uint16 [blend channel input], [Additive Tag] => [ { 50.f, 1} ]
37 * Outputs:
38 * float [float component], uint16 [blend channel output] => [ { 0.f, 0}, { 0.f, 1} ]
39 * To perform blending for this data, accumulation buffers are allocated per-blend-type, and each blend accumulates
40 * into the index of its blend channel input component. A final combination pass walks over blend channel outputs
41 * a writes the results into the result component from the accumulation buffer.
42 */
