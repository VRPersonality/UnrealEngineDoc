13/**
14 * System that maintains a FScopedMovementUpdate for any USceneComponent that has an
15 * animated transform or attachment for the duration of the evaluation to avoid repeatedly
16 * updating child transforms and/or overlaps.
17 *
18 * This system can be enabled/disabled using the cvar Sequencer.DeferMovementUpdates (is disabled by default)
19 * Output logging options are controlled by Sequencer.OutputDeferredMovementMode which is useful for tracking
20 * how many components are being moved in any given frame.
21 *
22 * This system runs in the Initialization and Evaluaion phases:
23 * Initialization: Gathers and initializes deferred movement for any AttachParent components that need (un)link
24 * Evaluation: Gathers and initializes deferred movement for any Component Transforms; queues up an event trigger to be executed at the end of the phase
25 * Finalization: Applys all pending movement updates in reverse order
26 */
