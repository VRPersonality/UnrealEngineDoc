37/**
38 * Structure used for movie scene evaluation templates contained within a track. Typically these are defined as one per-section.
39 * Serialized into a FMovieSceneEvaluationTemplate contained within the sequence itself (for fast initialization at runtime).
40 * Templates are executed in a 3-phase algorithm:
41 * 1) Initialize: (opt-in) Called at the start of the frame. Able to access mutable state from the playback context. Used to initialize any persistent state required for the evaluation pass.
42 * 2) Evaluate: Potentially called on a thread. Should (where possible) perform all costly evaluation logic, accumulating into execution tokens which will be executed at a later time on the game thread.
43 * 3) Execute: Called on all previously submitted execution tokens to apply the evaluated state to the movie scene player
44 */
