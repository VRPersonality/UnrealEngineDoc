21/**
22 * UMultiClickSequenceInputBehavior implements a generic multi-click-sequence input behavior.
23 * For example this behavior could be used to implement a multi-click polygon-drawing interaction.
24 *
25 * The internal state machine works as follows:
26 * 1) on input-device-button-press, check if target wants to begin sequence. If so, begin capture.
27 * 2) on button *release*, check if target wants to continue or terminate sequence
28 * a) if terminate, release capture
29 * b) if continue, do nothing (capture continues between presses)
30 *
31 * The target will receive "preview" notifications (basically hover) during updates where there is
32 * not a release. This can be used to (eg) update a rubber-band selection end point
33 *
34 * @todo it may be better to implement this as multiple captures, and use hover callbacks to
35 * do the between-capture previews. holding capture across mouse release is not ideal.
36 */
