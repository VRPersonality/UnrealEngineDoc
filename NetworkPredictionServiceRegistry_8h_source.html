15// Services do the actual system work on batches of registered instances. UNetworkPredictionWorldManager orchestrates them.
16//
17// Services should treat all registered instances the same in most cases. Instead of 1 services that has different behavior based
18// on Role/NetConnection/etc, make N services to cover each unique case. There will be exceptions to this where a role branch on a minor
19// aspect of the service is better than creating a brand new service.
20//
21// Services are defined by an interface (E.g, IFixedRollbackService) and a ModelDef based template implementation (E.g, TFixedRollbackService).
22//
23// Services operate on the data stored in TModelDataStore, which is owned by the FNetworkPredictionServiceRegistry.
24// All services get a pointer to the data store when created.
25// Services are free to keep internal acceleration structures but should mainly operate on the per-instance data in the data store.
26//
27// FNetworkPredictionServiceRegistry maintains lists of all active services. Services are bound to the ENetworkPredictionService enum for identification.
28// The template implementations are instantiated on demand during registration where ModelDef::ID is the index into the TServiceStorage array.
29//
30// The big picture looks like this, where 1=templated class has been instantiated and is managing X registered instances:
31//
32// [ModelDef::ID]
33// [ServiceType] 0 1 2 3 4 5 ... N
34// FixedRollback 1 0 1 1 1 0
35// FixedExtrapolated 0 1 0 0 1 0
36// ...
37// IndependentFinalize 0 0 0 0 0 1
38//
39//
40// NPs instance are registered to X services based on their config and network role/connection. Subscribed services are stored in TInstanceData<ModelDef>::ServiceMask.
41// As runtime settings change, E.g, NetworkLOD, the set of subscribed services will change. This is done efficiently with the ServiceMask.
42//
43// Services are an implementation detail of the UNetworkPredictionWorldManager and should not be exposed directly to outside runtime code.
44// E.g, don't pass pointers to services back to user code.
45//
46// Services can be specialized on ModelDef types. This could facilitate more efficient batch processing or further per-simulation/modeldef customization.
47// Services should have RegisterInstance/UnregisterInstance functions that take only the FNetworkPredictionID as parameter.
48//
49// No direct cross-service communication. Services can have their own internal services (E.g, rollback has an internal tick service) but the "top level" services do not communicate.
50// UNetworkPredictionWorldManager should coordinate things.
51//
52//
53// Adding new services:
54// 1. Add entry to ENetworkPredictionService
55// 2. Add NP_DECLARE_SERVICE
56// 3. Add NP_DEFINE_SERVICE_CALL
57// 4. Add logic to UNetworkPredictionWorldManager::ConfigureInstance to determine the conditions for subscribing to the service
58// 5. Add logic in UNetworkPredictionWorldManager to actually invoke the service. This will obviously be service dependent.
59//
60// New services types are not meant to be arbitrarily added by projects. Adding a brand new service requires modification of UNetworkPredictionWorldManager.
61// If you want to tack on 'something special', look at specializing an existing service (E.g the tick service could do 'extra stuff' per tick for example).
62//
63// Future Improvements:
64// -All services template classes are currently instantiated for all ModelDefs (the templated code is generated for each ModelDef)
65// -Even if ModelDefs are incompatible for a given service. E.g, a pure physics ModelDef with no Simulation still has the TLocalInputService<PhysicsModelDef> code instantiated.
66// -To be clear: it does not instantiate an instance of TLocalInputService at runtime, but the code is generated and almost certainly cannot be culled by the compiler.
67// -Concepts could be used to determine if ModelDefs are compatible with Services and we could avoid the template code instantiation.
68// -This would add more template machinery, and basically requires all TService<ModelDefs> be hidden behind SFINAE walls. E.g, Factories for instantiating and CallOrNot for Register/Unregister.
69// -Its not clear if its worth the effort at this point. In practice there should be relatively few ModelDefs that would benefit from this.
70// -But it could make a difference in some cases for build time and exe size

80 // Registers instance with given services. Will unregister instance with any previously-subscribed services.
81 template<typename ModelDef>
82 void RegisterInstance(FNetworkPredictionID ID, TInstanceData<ModelDef>& InstanceData, ENetworkPredictionService ServiceMask)
83 {
84 // Expected to register for fixed XOR independent services
85 npEnsureSlow(EnumHasAnyFlags(ServiceMask, ENetworkPredictionService::ANY_FIXED) ^ EnumHasAnyFlags(ServiceMask, ENetworkPredictionService::ANY_INDEPENDENT));
86 if (InstanceData.ServiceMask != ENetworkPredictionService::None)
87 {
88 // Only unregister/register what is changing
89 const ENetworkPredictionService UnregisterMask = InstanceData.ServiceMask & ~(ServiceMask);
90 const ENetworkPredictionService RegisterMask = ServiceMask & ~(InstanceData.ServiceMask);
91
92 UnregisterInstance_Internal<ModelDef>(ID, UnregisterMask);
93
94 ForEachService<ModelDef>(RegisterMask, [ID](auto Ptr)
95 {
96 Ptr->RegisterInstance(ID);
97 });
98 }
99 else
100 {
101 // Register with everything
102 ForEachService<ModelDef>(ServiceMask, [ID](auto Ptr)
103 {
104 Ptr->RegisterInstance(ID);
105 });
106 }
107
108 InstanceData.ServiceMask = ServiceMask;
109 }

216 template<ENetworkPredictionService ServiceMask, typename ServiceType, typename FuncRefType, typename ServiceStorageType>
217 void ConditionalCallFuncOnService(ServiceStorageType& ServiceStorage, const FuncRefType& Func, ENetworkPredictionService Mask)
218 {
219 using ModelDef = typename ServiceType::ModelDef;
220 if (EnumHasAllFlags(Mask, ServiceMask))
221 {
222 // Resize array for this ModelDef if necessary
223 if (ServiceStorage.Array.IsValidIndex(ModelDef::ID) == false)
224 {
225 FSparseArrayAllocationInfo AllocationInfo = ServiceStorage.Array.InsertUninitialized(ModelDef::ID);
226 new (AllocationInfo.Pointer) TUniquePtr<typename ServiceStorageType::ServiceInterface>();
227 }
228
229 // Allocate instance on the UniquePtr if necessary
230 auto& Ptr = ServiceStorage.Array[ModelDef::ID];
231 if (Ptr.IsValid() == false)
232 {
233 Ptr = MakeUnique<ServiceType>(GetDataStore<ModelDef>());
234 }
235
236 Func((ServiceType*)Ptr.Get());
237 }
238 }

240 template<typename ServiceType, typename ArrayType>
241 ServiceType* GetService_Internal(ArrayType& Array)
242 {
243 using ModelDef = typename ServiceType::ModelDef;
244 if (Array.IsValidIndex(ModelDef::ID) == false)
245 {
246 FSparseArrayAllocationInfo AllocationInfo = Array.InsertUninitialized(ModelDef::ID);
247 new (AllocationInfo.Pointer) TUniquePtr<ServiceType>(new ServiceType(GetDataStore<ModelDef>()));
248 }
249
250 auto& Item = Array[ModelDef::ID];
251 npCheckf(Item.IsValid(), TEXT("Service not initialized"));
252 return (ServiceType*)Item.Get();
253 }
