2697/**

2699 * are "mapped" or "assigned" to local players in your game. Depending on this
2700 * policy, the creation of FPlatformUserId's and FInputDeviceId's can change.
2701 *
2702 * This policy is only in affect on platforms which do NOT have a managed user login
2703 * or some kind of dedicated input mapping OS functionality (consoles).
2704 */
2705UENUM()
2706enum class EInputDeviceMappingPolicy : int32
2707{
2708 /**
2709 * Invalid mapping policy. Used to validate that the one in the settings is correct.
2710 */
2711 Invalid = -1 UMETA(Hidden),
2712
2713 /**
2714 * Use the current build target's built in platform login functionality.
2715 * This is often times a prompt which is displayed by the target system
2716 * to assign an input device to a specific player.
2717 */
2718 UseManagedPlatformLogin = 0,
2719
2720 /**
2721 * Maps the keyboard/mouse and the first connected controller to the primary platform user.
2722 * Any subsequently connected gamepads will be mapped to the next platform user without a
2723 * valid input device already assigned to them.
2724 *
2725 * An example of this policy when you have 3 gamepads and a keyboard/mouse connected would be something like this;
2726 *
2727 * -- Platform User 0 (Primary)
2728 * ---> Keyboard and mouse
2729 * ---> Gamepad 0
2730 * |
2731 * -- Platform User 1
2732 * ---> Gamepad 1
2733 * |
2734 * -- PlatformUser 2
2735 * ---> Gamepad 2
2736 */
2737 PrimaryUserSharesKeyboardAndFirstGamepad = 1,
2738
2739 /**
2740 * If this policy is set, then every input device that is connected will be mapped
2741 * to a unique platform user, meaning that it is treated as a separate local player.
2742 * This is what you want if you want plugging in a new controller to mean that you have a
2743 * new local player in your game.
2744 *
2745 * This only defines the default behavior upon connection of a new device, you can still remap
2746 * these input devices to new platform users later in the application lifecycle.
2747 */
2748 CreateUniquePlatformUserForEachDevice = 2,
2749
2750 /**
2751 * If this policy is set, then every connected input device will be mapped to the
2752 * primary platform user and new platform user Id's will not be created from input
2753 * devices.
2754 */
2755 MapAllDevicesToPrimaryUser = 3
