65 // This structures is the header for a node's instance data. The memory layout is as follows:
66 //
67 // [FNodeInstance] for the header
68 // [FTraitInstanceData] for trait 1
69 // [FTraitInstanceData] for trait 2
70 // [FTraitInstanceData] for trait 3
71 // [...]
72 // [LatentProperty] for some trait
73 // [LatentProperty] for some trait
74 // [...]
75 //
76 // Each node is thus followed by the trait instance data contiguously.
77 // Following the trait instance data, cached latent properties are present
78 //
79 // Each trait contains an instance data structure that derives from FTraitInstanceData.
80 // That derived structure is what is contained in the actual buffer. As such, sizes and offsets
81 // vary as required. The [FTraitTemplate] contains the offsets that map here.
82 //
83 // Optional padding is inserted as required by alignment constraints.
