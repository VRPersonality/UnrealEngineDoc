24/**
25 * FObjectRef represents a heavyweight reference that contains the specific pieces of information needed to reference an object
26 * (or null) that may or may not be loaded yet. The expectation is that given the imports of a package we have loaded, we should
27 * be able to create an FObjectRef to objects in a package we haven't yet loaded. For this reason, FObjectRef has to be derivable
28 * from the serialized (not transient) contents of an FObjectImport.
29 */
