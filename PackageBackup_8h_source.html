15/**
16 * Class that houses various static utility functions to allow for packages
17 * to be backed-up to a special auto save/backup directory. Backups are traditionally
18 * made before a save occurs to a package, to help guard against corruption/data loss.
19 */

50 /**
51 * Helper function designed to determine if the provided package should be backed up or not.
52 * The function checks for many conditions, such as if the package is too large to backup,
53 * if the package has a particular attribute that should prevent it from being backed up (such
54 * as being marked for PIE-use), if cooking is in progress, etc.
55 *
56 * @param InPackage Package which should be checked to see if its valid for backing-up

72 /**

80 /**

84 *
85 * @return The interval to wait before allowing another backup of the same package, in seconds
86 */
87 static int32 GetBackupInterval();
88
89 /**

92 * @return String containing the directory to store package backups in.

96 /**
97 * Deletes old backed-up package files until the provided amount of space (in bytes)
98 * is available to use in the backup directory. Fails if the provided amount of space
99 * is more than the amount of space the user has allowed for backups or if enough space
100 * could not be made.
101 *
102 * @param InBackupFiles File info of the files in the backup directory
103 * @param InSpaceUsed The amount of space, in bytes, the files in the provided array take up
104 * @param InSpaceRequired The amount of space, in bytes, to assure is available
105 * for the purposes of package backups
106 *
107 * @return true if the space was successfully provided, false if not or if the requested space was
108 * greater than the max allowed space by the user
109 */
110 static bool PerformBackupSpaceMaintenance( TArray<struct FBackupFileInfo>& InBackupFiles, int64 InSpaceUsed, int64 InSpaceRequired );
