14/*
15* Passthrough Blend :
16* It will generate an output using the base trait input keyframe and the selected alpha blend type (float, bool or curve)
17* Requires a base trait with a single input and PostEvaluate task creation (currently used to blend Control Rig Trait)
18*
19* If alpha type is Float or Bool
20* Depending on the weight value
21* - 0 (not relevant) : The input "as is" will be the output (passthrough). No call to PostEvaluate on the base will be performed.
22* - 1 (full weight) : The output of the base will be used as the output without blending by calling base PostEvaluate
23* - Any other relevant weight : Input will be duplicated (with a task) in Post Evaluate
24* The base Post Evaluate will be called to execute on the copy (normally will create a task)
25* Both keyframes (original and copy with base modifications) will be blended using a two keyframes blend task, passing the calculated alpha blend weight
26*
27* If alpha type is Curve
28* The curve value is not known until we are in the task (because we need the keyframe to extract the curve value)
29* Currently, the process will generate a full blend : A task will added to duplicate input keyframe at PostEvaluate
30* Base PostEvaluate will be called to allow task creation
31* Resulting keyframes will be blended with a two keyframes blend task, passing the curve data so the alpha can be obtained
32* If we try to get current alpha value of a Curve blend type, the returned value will be 1
33*/
