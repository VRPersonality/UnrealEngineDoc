20/**
21 * USingleClickInputBehavior implements a standard "button-click"-style input behavior.
22 * The state machine works as follows:
23 * 1) on input-device-button-press, hit-test the target. If hit, begin capture
24 * 2) on input-device-button-release, hit-test the target. If hit, call Target::OnClicked(). If not hit, ignore click.
25 *
26 * The second hit-test is required to allow the click to be "cancelled" by moving away
27 * from the target. This is standard GUI behavior. You can disable this second hit test
28 * using the .HitTestOnRelease property. This is strongly discouraged.
29 *
30 * The hit-test and on-clicked functions are provided by a IClickBehaviorTarget instance.
31 */
