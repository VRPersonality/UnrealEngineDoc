24/**
25 * USingleClickOrDragInputBehavior is a combination of a USingleClickBehavior and UClickDragBehavior,
26 * and allows for the common UI interaction where a click-and-release does one action, but if the mouse
27 * is moved, then a drag interaction is started. For example click-to-select is often combined with
28 * a drag-marquee-rectangle in this way. This can be directly implemented with a UClickDragBehavior but
29 * requires the client to (eg) detect movement thresholds, etc. This class encapsulates all that state/logic.
30 *
31 * The .ClickDistanceThreshold parameter determines how far the mouse must move (in whatever device units are in use)
32 * to switch from a click to drag interaction
33 *
34 * The .bBeginDragIfClickTargetNotHit parameter determines if the drag interaction will be immediately initiated
35 * if the initial 'click' mouse-down does not hit a valid clickable target. Defaults to true.
36 *
37 * The hit-test and on-clicked functions are provided by a IClickBehaviorTarget instance, while an
38 * IClickDragBehaviorTarget provides the can-click-drag/begin-drag/update-drag/end-drag functionality.
39 */
