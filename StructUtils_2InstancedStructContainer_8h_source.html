13/**
14 * Array of heterogeneous structs. Can be used as a property, supports serialization,
15 * but does not have type customizations (no editing in the UI).
16 *
17 * If you need UI editable array of heterogeneous structs, use TArray<FInstancedStruct> instead.
18 *
19 * The array item values and the index to an item are stored in one contiguous block of memory.
20 * The size required to specific layout of structs is larger than the sum of their sizes due to alignment,
21 * and because the index to the structs is stored along with the value memory. Each item takes extra 16 bytes
22 * for index. If your items are roughly same size, a TArray<TVariant<>> might be more performant.
23 *
24 * Adding new items is more expensive than on regular TArray<>, layout of the structs needs to be updated,
25 * and initialization is done via UScriptStruct. Adding and removing items should be done in chunks if possible.
26 *
27 * The allocation of new items does not allocate extra space as most array implementations do.
28 * Use Reserve() to reserve certain sized buffer in bytes if that is applicable to your use case.
29 */
