20/**
21 * A tool target is a stand-in object that a tool can operate on. It exposes the necessary
22 * interfaces to the tool.
23 *
24 * There are two intended purposes of the tool target system:
25 * 1. Allow tools to operate on arbitrary objects as long as they can be made to provide the
26 * tool with the necessary inputs. For instance, a mesh editing tool should be able to operate
27 * on skeletal, static, volume, and other mesh as long as the target manager has a registered
28 * factory that can use that type of mesh to create a suitable target.
29 * 2. (not yet used) Help cache tool inputs. I.e., if a tool requires an expensive
30 * conversion before it can work on an item, the converted result can be stored in the
31 * tool target which can be cached by the target manager and provided the next time the
32 * same type of target is requested for that item.
33 *
34 * Given an object, tool builders usually ask the target manager to turn it into a target that
35 * has the interfaces the tools needs. The tools cast the target to those interfaces to use
36 * them.
37 */
