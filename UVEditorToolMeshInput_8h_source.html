22/**
23 * A package of the needed information for an asset being operated on by a
24 * UV editor tool. It includes a UV unwrap mesh, a mesh with the UV layer applied,
25 * and background-op-compatible previews for each. It also has convenience methods
26 * for updating all of the represenations from just one of them, using a "fast update"
27 * code path when possible.
28 *
29 * This tool target is a bit different from usual in that it is not created
30 * by a tool target manager, and therefore doesn't have an accompanying factory.
31 * Instead, it is created by the mode, because the mode has access to the worlds
32 * in which the previews need to be created.
33 *
34 * It's arguable whether this should even inherit from UToolTarget.
35 */
36UCLASS()

139 // Notes above the below convenience functions:
140 // 1. Passing nullptr for ChangedVids/ChagnedElementIDs/ChangedConnectivityTids means that all vids/elements/tids
141 // need updating, respectively. Passing UUVEditorToolMeshInput::NONE_CHANGED_ARG is equivalent to passing a pointer
142 // to an empty array, and means that none changed. Otherwise only the vids/elements/tids in the pointed-to arrays
143 // are iterated over in the updated mesh.
144 // While passing these allows updates to be quicker, note that failing to include the relevant tids/vids/elements
145 // can put the input object into an invalid state or hit a check, for instance if a triangle refers to a new vid
146 // that was not added in.
147 // 2. ChangedVids/ChangedElementIDs is allowed to have new elements, since it is natural to split UVs. However,
148 // ChangedConnectivityTids must not have new Tids, because the tids form our correspondence between the unwrap mesh and
149 // the original layer.
150 // Also note that if you are gathering the changed vids/elements based off of triangles, you should gather them
151 // from the post-change mesh/overlay so that you do not miss any added elements, and since removed elements are
152 // captured by changed tri connectivity.
153 // 3. FastRenderUpdateTids is an optional list of triangles whose render data needs updating, which can allow for faster preview
154 // updates if the render buffers are properly split apart. If provided, it should be a superset of ChangedConnectivityTids,
155 // and should contain the one-ring triangles of ChangedVids.
156 // 4. If updating the preview objects, note that the functions do not try to cancel any active computations, so an active
157 // computation could reset things once it completes.
158
159 // Can be passed in as a ChangedVids/ChangedElements/ChangedConnectivityTids arguments as an equivalent of
160 // passing a pointer to an empty array.
161 static const TArray<int32>* const NONE_CHANGED_ARG;
