15/*
16 This is a wrapper around a native object for us to know what fields have been initialized or not in the object.
17 We only need this for native objects. For pure-Verse objects, we rely on checking if the `VValue` itself in the
18 object is/isn't initialized. But native objects don't always have a 1:1 `VObject` representation, and the data
19 lives in the `UObject` itself, so that approach doesn't work.
20
21 To handle this, we wrap native objects in a `VNativeConstructorWrapper` object that contains a bitmap of the fields
22 initialized instead, and keep track of the fields that way instead. When we are done with archetype construction, we
23 unwrap the native object using a special opcode and return it as part of `NewObject`, and let the wrapper object get
24 GC'ed during the next collection.
25
26 By convention, non-native Verse objects are not wrapped; the unwrap opcode just no-ops and returns the
27 object itself when it encounters it (this is so we can avoid allocating an extra wrapper object in the common
28 non-native case).
29
30 Therefore, we always favour emitting the unwrap instruction where a native object needs to be unwrapped, since we
31 don't know during codegen whether the object in question is native or not (since we do the wrapping in `NewObject` at
32 runtime). Non-native objects can also be turned into native objects at any time (we test this as well using
33 `--uobject-probability` in our tests.)
34 */
