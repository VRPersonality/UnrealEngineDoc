51 /*
52 * Mutable variables store their data as a `VRestValue`.
53 * It's not an array of `VValue`s because you can potentially load a class member before actually defining it. i.e.
54 *
55 * ```
56 * c := class {x:int}
57 * C := c{}
58 * Foo(C.X) # allocates a placeholder
59 * C.X := 1 # This is the first time `c.X` actually gets defined.
60 * ```
61 *
62 * This stores the actual data for individual fields. Some constants and procedures are stored in the shape, not the
63 * object (since then there's no need to do an unnecessary index lookup).
64 *
65 * The mapping of offsets to each field are stored in the emergent type's "shape". The reason why the object
66 * doesn't just store the mapping of fields to data itself is that it will eventually help when we implement inline
67 * caches for retrieving fields on objects. It also helps reduce memory usage because multiple objects can share
68 * the same hash table that describes their layouts.
69 */
70 inline void* GetData(const VCppClassInfo& CppClassInfo) const;
71 inline VRestValue* GetFieldData(const VCppClassInfo& CppClassInfo) const;
72};
