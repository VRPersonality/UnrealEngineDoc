◆ Lock

FRWLock FRedirectionSummary::Lock
private

CoreRedirects are written when the engine is single threaded, but we do not write the hashes for the global and per-package containers until they are requested on demand.

The requests on demand can occur on multiple threads, so we need a lock to ensure that the first thread that requests them and finds them dirty does not cause a data race with the second thread requests them.
