Detailed Description

Users of the Tools Framework need to implement IToolsContextTransactionsAPI so that the Tools have the ability to create Transactions and emit Changes.

Note that this is technically optional, but that undo/redo won't be supported without it.

Forward an invalidation request from Tools framework, to cause repaint/etc.

This is not always necessary but in some situations (eg in Non-Realtime mode in Editor) a redraw will not happen every frame. See UInputRouter for options to enable auto-invalidation.
