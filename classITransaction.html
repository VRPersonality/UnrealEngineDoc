Detailed Description

Interface for transactions.

Transactions are created each time an UObject is modified, for example in the Editor. The data stored inside a transaction object can then be used to provide undo/redo functionality.

virtual bool ITransaction::IsTransient ( ) const
pure virtual

Report if a transaction should be put in the undo buffer.

A transaction will be transient if it contains PIE objects or result in a no-op. If this returns true the transaction won't be put in the transaction buffer.

Returns
true if the transaction is transient.

virtual void ITransaction::StoreUndo ( UObject * Object,
TUniquePtr< FChange > CustomChange )
pure virtual

Stores a command that can be used to undo a change to the specified object.

This may be called multiple times in the same transaction to stack up changes that will be rolled back in reverse order. No copy of the object itself is stored.

Parameters
ObjectThe object the undo change will apply to
CustomChangeThe change that can be used to undo the changes to this object.
