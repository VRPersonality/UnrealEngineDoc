◆ ECorrespondenceStrategy

enum class UE::Geometry::FMeshImageBakingCache::ECorrespondenceStrategy
strong

ECorrespondenceStrategy determines the basic approach that will be used to establish a mapping from points on the BakeTarget Mesh (usually low-poly) to points on the Detail Mesh (eg highpoly).

Geometrically this is not a 1-1 mapping so there are various options

Enumerator
RaycastStandard 

Raycast inwards from Point+Thickness*Normal, if that misses, try Outwards from Point, then Inwards from Point.

NearestPoint 

Use geometrically nearest point.

Thickness is ignored

RaycastStandardThenNearest 

Use RaycastStandard but fall back to NearestPoint if none of the rays hit.

Identity 

Assume that BakeTarget == DetailMesh and so no mapping is necessary.

◆ FindSamplingHoles()

DYNAMICMESH_API void UE::Geometry::FMeshImageBakingCache::FindSamplingHoles ( TFunctionRef< bool(const FVector2i &)> IsValidSampleFunction,
TArray< FVector2i > & HolePixelsOut,
bool bParallel = true ) const

Iterate over valid samples in the Occupancy map to find "holes", ie pixels where no valid sample could be computed according to IsValidSampleFunction.

This can be used after a bake to find locations where the baked map needs to be repaired as a post-process (eg using image infilling/etc), which can produce better results than (eg) using a "worse" sampling strategy to guarantee some non-garbage result (eg Nearest-point sampling)
