Detailed Description

Describe the state list used to get to a specific state.

Since a state can be a subtree and can be linked to other states, the path to activate the subtree can be different. A state can enter, then exit, then reenter. It is considered the same state path but not the same unique state path. For a tree like RootA(0) StateA(1) StateB(2) LinkedStateA(3) StateC(4) LinkedStateA(5) SubTreeA(6) StateD(7) The path to get to StateD can be RootA.StateB.LinkedStateA.SubTreeA.StateD(0.2.3.6.7)
