14/**
15 * SCENE VIEW EXTENSIONS
16 * -----------------------------------------------------------------------------------------------
17 *
18 * This system lets you hook various aspects of UE rendering.
19 * To create a view extension, it is advisable to inherit
20 * from FSceneViewExtensionBase, which implements the
21 * ISceneViewExtension interface.
22 *
23 *
24 *
25 * INHERITING, INSTANTIATING, LIFETIME
26 * -----------------------------------------------------------------------------------------------
27 *
28 * In order to inherit from FSceneViewExtensionBase, do the following:
29 *
30 * class FMyExtension : public FSceneViewExtensionBase
31 * {
32 * public:
33 * FMyExtension( const FAutoRegister& AutoRegister, FYourParam1 Param1, FYourParam2 Param2 )
34 * : FSceneViewExtensionBase( AutoRegister )
35 * {
36 * }
37 * };
38 *
39 * Notice that your first argument must be FAutoRegister, and you must pass it
40 * to FSceneViewExtensionBase constructor. To instantiate your extension and register
41 * it, do the following:
42 *
43 * FSceneViewExtensions::NewExtension<FMyExtension>(Param1, Param2);
44 *
45 * You should maintain a reference to the extension for as long as you want to
46 * keep it registered.
47 *
48 * TSharedRef<FMyExtension,ESPMode::ThreadSafe> MyExtension;
49 * MyExtension = FSceneViewExtensions::NewExtension<FMyExtension>(Param1, Param2);
50 *
51 * If you follow this pattern, the cleanup of the extension will be safe and automatic
52 * whenever the `MyExtension` reference goes out of scope. In most cases, the `MyExtension`
53 * variable should be a member of the class owning the extension instance.
54 *
55 * The engine will keep the extension alive for the duration of the current frame to allow
56 * the render thread to finish.
57 *
58 *
59 *
60 * OPTING OUT of RUNNING
61 * -----------------------------------------------------------------------------------------------
62 *
63 * Each frame, the engine will invoke ISceneVewExtension::IsActiveThisFrame() to determine
64 * if your extension wants to run this frame. Returning false will cause none of the methods
65 * to be called this frame. The IsActiveThisFrame() method will be invoked again next frame.
66 *
67 * If you need fine grained control over individual methods, your IsActiveThisFrame should
68 * return `true` and gate each method as needed.
69 *
70 *
71 *
72 * PRIORITY
73 * -----------------------------------------------------------------------------------------------
74 * Extensions are executed in priority order. Higher priority extensions run first.
75 * To determine the priority of your extension, override ISceneViewExtension::GetPriority();
76 *
77 */
