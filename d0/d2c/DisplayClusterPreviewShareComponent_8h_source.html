57/**
58 * nDisplay Viewport preview share component
59 *
60 * It shares using Shared Memory Media the viewport textures of the parent nDisplay Actor.
61 * It should only be added to DisplayClusterRootActor instances, and only one component per instance.
62 * The way it works is that the sender generates a unique name for each viewport and captures its texture
63 * by getting a pointer to it from the corresponding Preview Component.
64 * The receiver will read it using the corresponding media source, and use the Texture Replace functionality
65 * in the nDisplay actor viewports to have them used and displayed.
66 */
