80/**
81 * Interface used by GetAssetRegistryTags to access the calling context data.
82 *
83 * API Notes:
84 * CookTag functions on this class add CookTags - tags that are added only when cooking and that are added to the
85 * development AssetRegistry used for patches patches, DLC, and Incremental cooking, but are removed from the runtime
86 * AssetRegistry passed into the game. CookTags can only be added when WantsCookTags() is true, calls in other Context
87 * states are ignored. All tags added with the CookTags interface have "Cook_" prepended to their names prior to being
88 * saved into the development to the asset registry.
89 *
90 * CookType: When GetAssetRegistryTags is called for package saves during cook, the cook type is set depending on
91 * the type of cook. Cooks in Unreal can be either CookByTheBook or CookOnTheFly. If GetAssetRegistryTags was called
92 * for other reasons, the CookType is reported as unknown. CookByTheBook means that the project's packaging settings
93 * are read by the cooker and a list of all referenced packages are found and those packages are saved and packaged.
94 * CookOnTheFly means that the game is running without packaged data, and is connecting to the cooker process to
95 * request the cooked version of packages that it demands. Only the packages requested by the runtime session are cooked.
96 */
