85/**
86 * The goal of this class is to provide an editor time version of BulkData that will work with the asset
87 * virtualization system.
88 *
89 * Assuming that the DDC is hot, the virtualized payloads are accessed relatively infrequently, usually when the package
90 * is being edited in the editor in some manner. So the payload access is designed around this. If the data is frequently
91 * accessed when running the editor then the user would not gain from having it virtualized as they would end up pulling
92 * it immediately anyway.
93 *
94 * The biggest difference with normal bulkdata is that the access times might be significantly longer if the
95 * payload is not readily available which is why the only way to access the payload is by a TFuture or a callback
96 * so that the caller is forced to consider how to handle the potential stall and hopefully organize their code
97 * in such a way that the time lag is not noticeable to the user.
98 *
99 * The second biggest difference is that the caller will own the payload memory once it is returned to them, unlike
100 * the old bulkdata class which would retain ownership. Which forces the calling code to be in control of when the
101 * memory is actually allocated and for how long. With the old bulkdata class a single access would leave that memory
102 * allocated in a non-obvious way and would lead to memory bloats within the editor.
103 *
104 * The method ::GetGuid can be used to access a unique identifier for the payload, currently it is based on the
105 * payload itself, so that two objects with the same payload would both have the same Guid. The intent is that we
106 * would be able to share local copies of the payload between branches to reduce the cost of having multiple branches
107 * with similar data on the same machine.
108 *
109 * Updating the bulkdata:
110 * Unlike the older bulkdata system, EditorBulkData does not support any lock/unlock mechanism, instead the internal
111 * data is updated via a single call to ::UpdatePayload and the data is retrieved via a single call to ::GetPayload.
112 *
113 * For examples see the "System.CoreUObject.Serialization.EditorBulkData.UpdatePayload" test found in
114 * Engine\Source\Runtime\CoreUObject\Private\Tests\Serialization\EditorBulkDataTests.cpp.
115 *
116 * Thread Safety:
117 *
118 * The class should be thread safe in that you can call the public methods on multiple threads at once and expect
119 * things to work. However if you were to call ::GetPayloadSize() before calling ::GetPayload() on one thread
120 * it would be possible for a call to ::UpdatePayload() to occur on another thread in between those two calls,
121 * meaning that the size you got might not be the correct size as the payload returned. If you need that level
122 * of thread safety it would be up to you to manage it at a higher level.
123 * It would however be safe to pass a copy of your FEditorBulkData to a background task rather than a reference.
124 * The copy would either be able to load the payload from disk, pull it from virtualized storage or if the
125 * source bulkdata is holding the payload in memory it will just take a reference to that payload rather than
126 * making a copy. In this way you can be sure of the state of the FEditorBulkData on the background task as
127 * future edits to the original bulkdata will not affect it.
128 */

376 /** Flags used to store additional meta information about the bulk data */
377 enum class EFlags : uint32
378 {
379 /** No flags are set */
380 None = 0,
381 /** Is the data actually virtualized or not? */
382 IsVirtualized = 1 << 0,
383 /** Does the package have access to a .upayload file? */
384 HasPayloadSidecarFile = 1 << 1,
385 /** The bulkdata object is currently referencing a payload saved under old bulkdata formats */
386 ReferencesLegacyFile = 1 << 2,
387 /** The legacy file being referenced is stored with Zlib compression format */
388 LegacyFileIsCompressed = 1 << 3,
389 /** The payload should not have compression applied to it. It is assumed that the payload is already
390 in some sort of compressed format, see the compression documentation above for more details. */
391 DisablePayloadCompression = 1 << 4,
392 /** The legacy file being referenced derived its key from guid and it should be replaced with a key-from-hash when saved */
393 LegacyKeyWasGuidDerived = 1 << 5,
394 /** (Transient) The Guid has been registered with the BulkDataRegistry */
395 HasRegistered = 1 << 6,
396 /** (Transient) The BulkData object is a copy used only to represent the id and payload; it does not communicate with the BulkDataRegistry, and will point DDC jobs toward the original BulkData */
397 IsTornOff = 1 << 7,
398 /** The bulkdata object references a payload stored in a WorkspaceDomain file */
399 ReferencesWorkspaceDomain = 1 << 8,
400 /** The payload is stored in a package trailer, so the bulkdata object will have to poll the trailer to find the payload offset */
401 StoredInPackageTrailer = 1 << 9,
402 /** The bulkdata object was cooked. */
403 IsCooked = 1 << 10,
404 /** (Transient) The package owning the bulkdata has been detached from disk and we can no longer load from it */
405 WasDetached = 1 << 11
406 };
