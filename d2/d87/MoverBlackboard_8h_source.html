21/** MoverBlackboard: this is a simple generic map that can store any type, used as a way for decoupled systems to
22 * store calculations or transient state data that isn't necessary to reconstitute the movement simulation.
23 * It has support for invalidation, which could occur, for example, when a rollback is triggered.
24 * Values submitted are copy-in, copy-out.
25 * Unlike a traditional blackboard pattern, there is no support for subscribing to changes.
26 * TODO: expand invalidation rules attached to BBObjs, for instance if we wanted some to invalidate upon rollback. Some might expire over time or after a number of simulation frames. Or an item could be tagged with a predicted sim frame #, and become cleared once that frame is finalized/confirmed.
27 */
