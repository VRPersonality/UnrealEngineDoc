12/**
13 * Custom property node that you'd like to expand pin by reflecting internal instance (we call TargetInstance here)
14 *
15 * Used by sub anim instance or control rig node
16 * where you have internal instance and would like to reflect to AnimNode as a pin
17 *
18 * To make pin working, you need storage inside of AnimInstance (SourceProperties/SourcePropertyNames)
19 * So this creates storage inside of AnimInstance with the unique custom property name
20 * and it copies to the actually TargetInstance here to allow the information be transferred in runtime (DestProperties/DestPropertyNames)
21 *
22 * TargetInstance - UObject derived instance that has certain dest properties
23 * Source - AnimInstance's copy properties that is used to store the data
24 */
