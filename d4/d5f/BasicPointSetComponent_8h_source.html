13/**
14 * UBasic2DPointSetComponent and UBasic3DPointSetComponent are components that draw a set of points as small squares. The 2D version stores XY coordinates with
15 * Z set to 0 whereas the 3D version stores XYZ coordinates.
16 * All points have the same color, size, and depth bias based on what is set via SetPointSetParameters(). The points are drawn as two triangles, i.e. a
17 * square, orthogonal to the view direction. The actual point size is calculated in the shader using a custom material.
18 *
19 * Before inserting any points via AddElement(), the maximum number of points that will be added needs to be declared via ReserveElements().
20 * The actual number of points that were added is available via NumElements().
21 *
22 * To support both 2D and 3D with minimal code duplication, the point set components are composed from the UBasicPointSetComponentBase class for the component
23 * specific functionality, and an instantiation of the TBasicElementSet template class for either FVector2f or FVector3f that encapsulates all point specific
24 * functionality. The final derived classes UBasic2DPointSetComponent and UBasic3DPointSetComponent contain identical boilerplate code to facilitate calls that
25 * need to go both to the component base class and the point base class, e.g. Clear() will both mark the component as dirty and delete all points.
26 */

58 /** Update component bounds using a provided function that determines the box around all point positions.
59 * This unifies how the bounds are calculated while keeping the calculation of the box around all point positions generic. */
60 template <typename CalcPointBoxFn>
61 FBoxSphereBounds UpdateComponentBounds(const FTransform& LocalToWorld, bool bPointsDirty, CalcPointBoxFn&& CalcPointBox) const
62 {
63 bBoundsDirty |= bPointsDirty;
64 if (bBoundsDirty)
65 {
66 // Calculating an FBox around a set of points is specific to the point type, and thus we call out to a provided type specific lambda.
67 const FBox Box = CalcPointBox();
68
69 Bounds = FBoxSphereBounds(Box);
70 bBoundsDirty = false;
71
72 // TODO: This next bit is not ideal because the point size is specified in onscreen pixels,
73 // so the true amount by which we would need to expand bounds depends on camera location, FOV, etc.
74 // We mainly do this as a hack against a problem in orthographic viewports, which cull small items
75 // based on their bounds, and a set consisting of a single point will always be culled due
76 // to having 0-sized bounds. It's worth noting that when zooming out sufficiently far, the
77 // point will still be culled even with this hack, however.
78 // The proper solution is to be able to opt out of the orthographic culling behavior, which is something
79 // we need to add.
80 if (Box.IsValid)
81 {
82 Bounds = Bounds.ExpandBy(Size);
83 }
84 }
85 return Bounds.TransformBy(LocalToWorld);
86 }
