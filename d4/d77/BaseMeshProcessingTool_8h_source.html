50/**
51 * UBaseMeshProcessingTool is a base Tool (ie has no functionality of it's own and must be subclassed)
52 * that provides the following structure:
53 * - a Background-Compute-With-Preview Temp Actor/Component is created based on the input mesh
54 * - The Subclass provides FDynamicMeshOperator instances (via IDynamicMeshOperatorFactory) that process/modify and update this Preview
55 * - PropertySets with custom visibility can be registered, and on change will invalidate the current computation
56 *
57 * Optional support for a WeightMap property set and tracking of active weight map can be enabled by calling
58 * AddWeightMapPropertySet(), GetActiveWeightMap() will then return the active WeightMap, and changes to the
59 * WeightMap selection will invalidate the computation.
60 *
61 * Most subclasses will only need to define their PropertySets and implement MakeNewOperator(), see eg SmoothMeshTool for a minimal example
62 *
63 * Other functions:
64 * - GetInitialMesh() : return reference to copy of initial mesh, used to initialize FDynamicMeshOperator
65 * - GetUPreviewMesh() : return the UPreviewMesh inside the background compute (for configuration/etc - should not directly touch the mesh!)
66 * - GetPreviewTransform() : return active FTransform on the Preview mesh, should be passed to FDynamicMeshOperator unless it is outputting world position
67 * - InvalidateResult() : subclasses call this to notify the base class that current result/computation has been invalidated
68 *
69 * The Base tool will do various optional precomputations or changes to the input mesh, which can be configured by
70 * overriding various functions below.
71 *
72 * RequiresInitialVtxNormals() : return true (default=false) to calculate per-vertex normals on the input mesh, returned by GetInitialVtxNormals()
73 *
74 * RequiresInitialBoundaryLoops() : return true (default=false) to calculate boundary loops on the input mesh, returned by GetInitialBoundaryLoops()
75 *
76 * RequiresScaleNormalization() : return true (default=true) to apply an initial scale to the input mesh so that it has consistent size
77 * before being sent into the computation. Scaling factor (eg for scaling UI constants) can be accessed via GetScaleNormalizationFactor()
78 *
79 */
