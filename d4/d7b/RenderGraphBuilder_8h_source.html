200 /** Adds a lambda pass to the graph with an accompanied pass parameter struct.
201 *
202 * RDG resources declared in the struct (via _RDG parameter macros) are safe to access in the lambda. The pass parameter struct
203 * should be allocated by AllocParameters(), and once passed in, should not be mutated. It is safe to provide the same parameter
204 * struct to multiple passes, so long as it is kept immutable. The lambda is deferred until execution unless the immediate debug
205 * mode is enabled. All lambda captures should assume deferral of execution.
206 *
207 * The lambda must include a single RHI command list as its parameter. The exact type of command list depends on the workload.
208 * For example, use FRHIComputeCommandList& for Compute / AsyncCompute workloads. Raster passes should use FRHICommandList&.
209 * Prefer not to use FRHICommandListImmediate& unless actually required.
210 *
211 * Declare the type of GPU workload (i.e. Copy, Compute / AsyncCompute, Graphics) to the pass via the Flags argument. This is
212 * used to determine async compute regions, render pass setup / merging, RHI transition accesses, etc. Other flags exist for
213 * specialized purposes, like forcing a pass to never be culled (NeverCull). See ERDGPassFlags for more info.
214 *
215 * The pass name is used by debugging / profiling tools.
216 */
