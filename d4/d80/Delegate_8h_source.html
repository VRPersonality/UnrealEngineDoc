14/**
15 * C++ DELEGATES
16 * -----------------------------------------------------------------------------------------------
17 *
18 * This system allows you to call member functions on C++ objects in a generic, yet type-safe way.
19 * Using delegates, you can dynamically bind to a member function of an arbitrary object,
20 * then call functions on the object, even if the caller doesn't know the object's type.
21 *
22 * The system predefines various combinations of generic function signatures with which you can
23 * declare a delegate type from, filling in the type names for return value and parameters with
24 * whichever types you need.
25 *
26 * Both single-cast and multi-cast delegates are supported, as well as "dynamic" delegates which
27 * can be serialized to disk and accessed from blueprints. Additionally, delegates may define
28 * "payload" data which will be stored and passed directly to bound functions.
29 *
30 *
31 *
32 * DELEGATE FEATURES
33 * -----------------------------------------------------------------------------------------------
34 *
35 * Currently we support delegate signatures using any combination of the following:
36 * - Functions returning a value
37 * - Up to four "payload" variables
38 * - Multiple function parameters depending on macro/template declaration
39 * - Functions declared as 'const'
40 *
41 * Multi-cast delegates are also supported, using the 'DECLARE_MULTICAST_DELEGATE...' macros.
42 * Multi-cast delegates allow you to attach multiple function delegates, then execute them all at
43 * once by calling a single "Broadcast()" function. Multi-cast delegate signatures are not allowed
44 * to use a return value.
45 *
46 * Unlike other types, dynamic delegates are integrated into the UObject reflection system and can be
47 * bound to blueprint-implemented functions or serialized to disk. You can also bind native functions,
48 * but the native functions need to be declared with UFUNCTION markup. You do not need to use UFUNCTION for
49 * functions bound to other types of delegates.
50 *
51 * You can assign "payload data" to your delegates! These are arbitrary variables that will be passed
52 * directly to any bound function when it is invoked. This is really useful as it allows you to store
53 * parameters within the delegate it self at bind-time. All delegate types (except for "dynamic") supports
54 * payload variables automatically!
55 *
56 * When binding to a delegate, you can pass payload data along. This example passes two custom variables,
57 * a bool and an int32 to a delegate. Then when the delegate is invoked, these parameters will be passed
58 * to your bound function. The extra variable arguments must always be accepted after the delegate
59 * type parameter arguments.
60 *
61 * MyDelegate.BindStatic( &MyFunction, true, 20 );
62 *
63 * Remember to look at the signature table at the bottom of this documentation comment for the macro names
64 * to use for each function signature type, and the binding table to see options and concerns for binding.
65 *
66 *
67 *
68 * DELEGATES EXAMPLE
69 * -----------------------------------------------------------------------------------------------
70 *
71 * Suppose you have a class with a method that you'd like to be able to call from anywhere:
72 *
73 * class FLogWriter
74 * {
75 * void WriteToLog( FString );
76 * };
77 *
78 * To call the WriteToLog function, we'll need to create a delegate type for that function's signature.
79 * To do this, you will first declare the delegate using one of the macros below. For example, here
80 * is a simple delegate type:
81 *
82 * DECLARE_DELEGATE_OneParam( FStringDelegate, FString );
83 *
84 * This creates a delegate type called 'FStringDelegate' that takes a single parameter of type 'FString'.
85 *
86 * Here's an example of how you'd use this 'FStringDelegate' in a class:
87 *
88 * class FMyClass
89 * {
90 * FStringDelegate WriteToLogDelegate;
91 * };
92 *
93 * This allows your class to hold a pointer to a method in an arbitrary class. The only thing the
94 * class really knows about this delegate is it's function signature.
95 *
96 * Now, to assign the delegate, simply create an instance of your delegate class, passing along the
97 * class that owns the method as a template parameter. You'll also pass the instance of your object
98 * and the actual function address of the method. So, here we'll create an instance of our 'FLogWriter'
99 * class, then create a delegate for the 'WriteToLog' method of that object instance:
100 *
101 * FSharedRef< FLogWriter > LogWriter( new FLogWriter() );
102 *
103 * WriteToLogDelegate.BindSP( LogWriter, &FLogWriter::WriteToLog );
104 *
105 * You've just dynamically bound a delegate to a method of a class! Pretty simple, right?
106 *
107 * Note that the 'SP' part of 'BindSP' stands for 'shared pointer', because we're binding to an
108 * object that's owned by a shared pointer. There are versions for different object types,
109 * such as BindRaw() and BindUObject(). You can bind to global function pointers with BindStatic().
110 *
111 * Now, your 'WriteToLog' method can be called by FMyClass without it even knowing anything about
112 * the 'FLogWriter' class! To call a your delegate, just use the 'Execute()' method:
113 *
114 * WriteToLogDelegate.Execute( TEXT( "Delegates are spiffy!" ) );
115 *
116 * If you call Execute() before binding a function to the delegate, an assertion will be triggered. In
117 * many cases, you'll instead want to do this:
118 *
119 * WriteToLogDelegate.ExecuteIfBound( TEXT( "Only executes if a function was bound!" ) );
120 *
121 * That's pretty much all there is to it!! You can read below for a bit more information.
122 *
123 *
124 *
125 * MORE INFORMATION
126 * -----------------------------------------------------------------------------------------------
127 *
128 * The delegate system understands certain types of objects, and additional features are enabled when
129 * using these objects. If you bind a delegate to a member of a UObject or shared pointer class, the
130 * delegate system can keep a weak reference to the object, so that if the object gets destroyed out
131 * from underneath the delegate, you'll be able to handle these cases by calling IsBound() or
132 * ExecuteIfBound() functions. Note the special binding syntax for the various types of supported objects.
133 *
134 * It's perfectly safe to copy delegate objects. Delegates can be passed around by value but this is
135 * generally not recommended since they do have to allocate memory on the heap. Pass them by reference
136 * when possible!
137 *
138 * Delegate signature declarations can exist at global scope, within a namespace or even within a class
139 * declaration (but not function bodies.)
140 *
141 *
142 *
143 * FUNCTION SIGNATURES
144 * -----------------------------------------------------------------------------------------------
145 *
146 * Use this table to find the declaration macro to use to declare your delegate.
147 * The full list is defined in DelegateCombinations.h
148 *
149 * Function signature | Declaration macro
150 * ------------------------------------------------------------------------------------------------------------------------------------------------------------
151 * void Function() | DECLARE_DELEGATE( DelegateName )
152 * void Function( <Param1> ) | DECLARE_DELEGATE_OneParam( DelegateName, Param1Type )
153 * void Function( <Param1>, <Param2> ) | DECLARE_DELEGATE_TwoParams( DelegateName, Param1Type, Param2Type )
154 * void Function( <Param1>, <Param2>, ... ) | DECLARE_DELEGATE_<Num>Params( DelegateName, Param1Type, Param2Type, ... )
155 * <RetVal> Function() | DECLARE_DELEGATE_RetVal( RetValType, DelegateName )
156 * <RetVal> Function( <Param1> ) | DECLARE_DELEGATE_RetVal_OneParam( RetValType, DelegateName, Param1Type )
157 * <RetVal> Function( <Param1>, <Param2> ) | DECLARE_DELEGATE_RetVal_TwoParams( RetValType, DelegateName, Param1Type, Param2Type )
158 * <RetVal> Function( <Param1>, <Param2>, ... ) | DECLARE_DELEGATE_RetVal_<Num>Params( RetValType, DelegateName, Param1Type, Param2Type, ... )
159 * ------------------------------------------------------------------------------------------------------------------------------------------------------------
160 *
161 * Remember, there are three different delegate types you can define (any of the above signatures will work):
162 *
163 * Single-cast delegates: DECLARE_DELEGATE...()
164 * Multi-cast delegates: DECLARE_MULTICAST_DELEGATE...()
165 * Dynamic (UObject, serializable) delegates: DECLARE_DYNAMIC_DELEGATE...()
166 *
167 *
168 *
169 * BINDING AND SAFETY
170 * -----------------------------------------------------------------------------------------------
171 *
172 * Once a delegate has been declared, it can be bound to functions stored in different places.
173 * Because delegates are often called long after they are bound, extra attention must be paid to
174 * avoid crashes. This list is for single-cast, for multi-cast delegates, replace Bind in the table
175 * below with Add. Also for multi-cast delegates, Add will return a handle that can then be used to
176 * later remove the binding. All multi-cast delegates have an ::FDelegate subtype defining an equivalent
177 * single-cast version, that can be Created one place and then added later.
178 *
179 * Bind function | Usage
180 * ------------------------------------------------------------------------------------------------------------------------------------------------------------
181 * BindStatic(&GlobalFunctionName) | Call a static function, can either be globally scoped or a class static
182 * BindUObject(UObject, &UClass::Function) | Call a UObject class member function via a TWeakObjectPtr, will not be called if object is invalid
183 * BindSP(SharedPtr, &FClass::Function) | Call a native class member function via a TWeakPtr, will not be called if shared pointer is invalid
184 * BindThreadSafeSP(SharedPtr, &FClass::Function) | Call a native class member function via a TWeakPtr, will not be called if shared pointer is invalid
185 * BindRaw(RawPtr, &FClass::Function) | Call a native class member function with no safety checks. You MUST call Unbind or Remove when object dies to avoid crashes!
186 * BindLambda(Lambda) | Call a lambda function with no safety checks. You MUST make sure all captures will be safe at a later point to avoid crashes!
187 * BindSPLambda(SharedPtr, Lambda) | Call a lambda function only if shared pointer is still valid. Captured 'this' will always be valid but any other captures may not be
188 * BindWeakLambda(UObject, Lambda) | Call a lambda function only if UObject is still valid. Captured 'this' will always be valid but any other captures may not be
189 * BindUFunction(UObject, FName("FunctionName")) | Usable for both native and dynamic delegates, will call a UFUNCTION with specified name
190 * BindDynamic(UObject, &UClass::FunctionName) | Convenience wrapper only available for dynamic delegates, FunctionName must be declared as a UFUNCTION
191 * ------------------------------------------------------------------------------------------------------------------------------------------------------------
192 *
193 */

468 /**
469 * Returns the root function name from a string representing a member function pointer.
470 * Note: this function only returns a pointer to the substring and doesn't create a new string.
471 *
472 * @param InMacroFunctionName The string containing the member function name.
473 * @return An FName of the member function name.
474 */
475 inline FName GetTrimmedMemberFunctionName(const TCHAR* InMacroFunctionName)
476 {
477 // We strip off the class prefix and just return the function name by itself.
478 check(InMacroFunctionName);
479 const TCHAR* Result = FCString::Strrstr( InMacroFunctionName, TEXT( "::" ) );
480 checkf(Result && Result[2] != (TCHAR)'0', TEXT("'%s' does not look like a member function"), InMacroFunctionName);
481 return FName(Result + 2);
482 }
