35/**
36 * Serves as a container for all available blueprint actions (no matter the
37 * type of blueprint/graph they belong in). The actions stored here are not tied
38 * to a specific ui menu; each action is a UBlueprintNodeSpawner which is
39 * charged with spawning a specific node type. Should be set up in a way where
40 * class specific actions are refreshed when the associated class is regenerated.
41 *
42 * @TODO: Hook up to handle class recompile events, along with enum/struct asset creation events.
43 */
