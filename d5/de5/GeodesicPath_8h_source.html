26/**

96/**
97* Given a single path initially defined by edges on a FDynamicMesh (the SurfaceMesh), this class uses an intrinsic mesh with
98* the same vertex set to minimize the length of the path on the surface mesh. This is done iteratively by a succession
99* of local path deformations comprised of (local) intrisic edge flips and path re-routing, each deformation locally reduces the path length.
100* The resulting path is comprised of intrinsic mesh edges.
101*
102* This is an implementation of the ideas in
103* "You Can Find Geodesic Paths in Triangle Meshes by Just Flipping Edges" by Nicholas Sharp and Keenan Crane
104* ACM Trans. Graph. Vol. 39, No. 6, 2020
105*
106* The original path is constructed as a sequence of mesh edges relative to the SurfaceMesh ( the FDynamicMesh3 ),
107* these edges are encoded with path direction as FDirectedSegments.
108* Note: The current implementation does not support closed paths or multiple paths.
109*/
110class FDeformableEdgePath
111{
112public:

138 /**
139 * Minimize the deformable edge path with respect to the total path length.
140 * This minimization finds a local minimum in path length but may not find the shortest over all distance between to points.
141 * In this sense the operation can be thought of as a path-straightening.
142 * @param DeformedPathInfo - on return this info struct is populated
143 * @param MaxNumIterations - the maximum number of two-edge joints that may be replaced by shorter paths.
144 */

163 * struct that references a point on a mesh, by vertex, by edge-crossing, or barycentric coords

166 /**
167 * @return an array of surface points relative to the SurfaceMesh that define this path.
168 * Note the first and last surfacepoints correspond to the start and end vertex, but all
169 * other surface points may be either edge crossings or vertex points.
170 *
171 * @param CoalesceThreshold - In barycentric units [0,1], edge-crossings within this threshold are snapped to the nearest vertex
172 * and any resulting repitition of vertex surface points are replaced with a single vertex surface point.
173 * Due to numerical precision issues, a path that 'should' intersect a surface mesh vertex may appear
174 * as a sequence of vertex adjacent edge-crossings very close to the vertex.. the threshold is applied in
175 * a post-process coalesce those crossings into a single vertex crossing.
176 */

244 * Traveling either CCW or CW around vertex CenterVID, from edge StartEID to edge EndEID, flip edges adjacent to CVID.
