19/** MetaSound Frontend Query
20 *
21 * MetaSound Frontend Query provides a way to systematically organize and update
22 * streaming data associated with the MetaSound Frontend. It is a streaming MapReduce
23 * framework for querying streams of data (https://en.wikipedia.org/wiki/MapReduce)
24 *
25 * While it does not support the computational parallelism commonly found in MapReduce
26 * frameworks, it does offer:
27 * - An encapsulated and reusable set of methods for manipulating streamed data.
28 * - Support for incremental updates (a.k.a. streamed data).
29 * - An indexed output for efficient lookup.
30 *
31 * Data within MetaSound Frontend Query is organized similarly to a NoSQL database
32 * (https://en.wikipedia.org/wiki/NoSQL). Each object (FFrontendQueryEntry) is
33 * assigned a unique ID. Keys (FFrontendQueryKey) are associated with sets of entries
34 * (FFrontendQueryPartition) and allow partitions to be retrieved efficiently.
35 * Each partition holds a set of entries which is determined by the steps in the
36 * query (FFrontendQuery). A FFrontendQueryKey or FFrontendQueryValue represent
37 * one of multiple types by using a TVariant<>.
38 *
39 * A query contains a sequence of steps that get executed on streaming data. The
40 * various types of steps reflect common operations performed in MapReduce and
41 * NoSQL database queries.
42 *
43 * Step Types
44 * Stream: Produce a stream of FFrontendQueryValues.
45 * Map: Map a FFrontendQueryEntry to a partition associated with a FFrontendQueryKey.
46 * Reduce: Apply an incremental summarization of a FFrontendQueryPartition.
47 * Transform: Alter a FFrontendQueryValue.
48 * Filter: Remove FFrontendQueryValues with a test function.
49 * Score: Calculate a score for a FFrontendQueryValue.
50 * Sort: Sort a FFrontendQueryPartition.
51 * Limit: Limit the size of a FFrontendQueryPartition.
52 */
