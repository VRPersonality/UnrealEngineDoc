55/**
56 * FGroupTopologyDeformer supports deforming a Mesh based on an overlaid FGroupTopology.
57 * First the client defines a set of "Handle" elements (Faces/Corners/Edges) using SetActiveHandleX().
58 * The client will provide new vertex positions for the vertices of these elements via the
59 * HandleVertexDeformFunc argument to UpdateSolution(). Once the Handle vertices have been
60 * updated, the deformer solves for updated vertex positions in the GroupTopology Faces
61 * that are adjacent to the handles. This region is referred to as the "ROI" (Region-of-Interest)
62 *
63 * The default deformation is to first solve for the updated edges, and then
64 * solve for updated faces. This is done via linear encoding of the edge and face vertices
65 * relative to their boundaries (edge boundary is endpoint corners, face boundary is edges).
66 *
67 * Various functions can be overrided to customize behavior.
68 *
69 */
