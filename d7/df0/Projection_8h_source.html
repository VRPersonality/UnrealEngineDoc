62/**
63 * Projection() is a related function to Invoke(), in that it can be used to invoke an object with a set of arguments.
64 * However, it works by transforming something invocable into something callable, i.e. can be called with a normal parenthesized set of arguments.
65 *
66 * // These are equivalent:
67 * Projection(I)(Args...)
68 * Invoke(I, Args...).
69 *
70 * ... with the same advantages:
71 *
72 * // Can accept member function pointers
73 * Invoke(&FObjType::Member, Obj); // equivalent to Obj.Member
74 * Invoke(&FObjType::Func, Obj, Args...); // equivalent to Obj.Func(Args...)
75 * Projection(&FObjType::Member)(Obj); // equivalent to Obj.Member
76 * Projection(&FObjType::Func)(Obj, Args...); // equivalent to Obj.Func(Args...)
77 *
78 * // Can operate on pointers, including smart pointers
79 * Invoke(&FObjType::Member, ObjPtr); // equivalent to ObjPtr->Member
80 * Invoke(&FObjType::Func, ObjPtr, Args...); // equivalent to ObjPtr->Func(Args...)
81 * Projection(&FObjType::Member)(ObjPtr); // equivalent to ObjPtr->Member
82 * Projection(&FObjType::Func)(ObjPtr, Args...); // equivalent to ObjPtr->Func(Args...)
83 *
84 * However, Projection() has some additional advantages:
85 *
86 * - Projection(I) returns I unchanged if it is already callable, meaning no redundant stepping in and out of many Invoke() calls in the debugger.
87 * - Projection(...) is variadic and can transform a sequence of invocables into a callable that invokes them one by one:
88 *
89 * Projection(A, B, C)(Args...)
90 * Invoke(C, Invoke(B, Invoke(A, Args...)))
91 *
92 * This allows users to pass a sequence of projections to an algorithm that takes a single projection:
93 *
94 * struct FInner
95 * {
96 * FString Name;
97 * };
98 * struct FOuter
99 * {
100 * FInner Inner;
101 * };
102 *
103 * // Sort array of outers by the names of inner,
104 * Algo::SortBy(ArrayOfOuters, Projection(&FOuter::Inner, &FInner::Name));
105 */
