9/**
10 * Behavior Tree service nodes is designed to perform "background" tasks that update AI's knowledge.
11 *
12 * Services are being executed when underlying branch of behavior tree becomes active,
13 * but unlike tasks they don't return any results and can't directly affect execution flow.
14 *
15 * Usually they perform periodical checks (see TickNode) and often store results in blackboard.
16 * If any decorator node below requires results of check beforehand, use OnSearchStart function.
17 * Keep in mind that any checks performed there have to be instantaneous!
18 *
19 * Other typical use case is creating a marker when specific branch is being executed
20 * (see OnBecomeRelevant, OnCeaseRelevant), by setting a flag in blackboard.
21 *
22 * Because some of them can be instanced for specific AI, following virtual functions are not marked as const:
23 * - OnBecomeRelevant (from UBTAuxiliaryNode)
24 * - OnCeaseRelevant (from UBTAuxiliaryNode)
25 * - TickNode (from UBTAuxiliaryNode)
26 * - OnSearchStart
27 *
28 * If your node is not being instanced (default behavior), DO NOT change any properties of object within those functions!
29 * Template nodes are shared across all behavior tree components using the same tree asset and must store
30 * their runtime properties in provided NodeMemory block (allocation size determined by GetInstanceMemorySize() )
31 */
