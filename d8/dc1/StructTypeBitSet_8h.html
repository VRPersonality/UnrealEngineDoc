We're declaring StructTracker this way rather than being a static member variable of TStructTypeBitSet to avoid linking issues.

We've run into all sorts of issues depending on compiler and whether strict mode was on, and this is the best way we could come up with to solve it. Thankfully the user doesn't need to even know about this class's existence as long as they are using the macros below.
