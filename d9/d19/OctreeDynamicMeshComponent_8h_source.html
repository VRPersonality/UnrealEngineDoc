22/**
23 * UOctreeDynamicMeshComponent is a mesh component similar to UProceduralMeshComponent,
24 * except it bases the renderable geometry off an internal FDynamicMesh3 instance.
25 * The class generally has the same capabilities as UDynamicMeshComponent.
26 *
27 * A FDynamicMeshOctree3 is available to dynamically track the triangles of the mesh
28 * (however the client is responsible for updating this octree).
29 * Based on the Octree, the mesh is partitioned into chunks that are stored in separate
30 * RenderBuffers in the FOctreeDynamicMeshSceneProxy.
31 * Calling NotifyMeshUpdated() will result in only the "dirty" chunks being updated,
32 * rather than the entire mesh.
33 *
34 * (So, if you don't need this capability, and don't want to update an Octree, use UDynamicMeshComponent!)
35 */
