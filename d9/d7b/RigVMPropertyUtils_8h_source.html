12/**
13 * Given a property, return the C++ type name for it and the associated type object. If the type is an unambiguous
14 * simple type, such as a bool or an int, the OutTypeObject value will be nullptr. If the property represents a
15 * ambiguous type, like an object pointer, interface or an enum, then the OutTypeObject will be the class of the
16 * pointed to object, or the enum definition. For container properties, the type object is always the contained type,
17 * rather than the type of the container.
18 * \param InProperty The property to get the type for.
19 * \param OutTypeName The C++ type name.
20 * \param OutTypeObject The object class, if the property type is complex or ambiguous.
21 */
22RIGVM_API void GetTypeFromProperty(const FProperty* InProperty, FName& OutTypeName, UObject*& OutTypeObject);

24/**
25 * Given a property and a value storage for it, return a 32-bit hash value for the underlying value of tha type.
26 * \note The resulting hash is not suitable for serialization, since the underlying hash combine function is not
27 * guaranteed to be stable between releases.
28 * \param InProperty The property to get a hash value for.
29 * \param InMemory The underlying memory that the property is defined for.
30 * \param InContainerType The type of memory being pointed at. For UObjects, use EPropertyPointerType::Container.
31 * If pointing directly at the value location in memory, use EPropertyPointerType::Direct.
32 * \return The hash value of the value of the property at the memory location the property lives at.
33 */
34RIGVM_API uint32 GetPropertyHashFast(const FProperty* InProperty, const uint8* InMemory, EPropertyPointerType InContainerType = EPropertyPointerType::Container);
