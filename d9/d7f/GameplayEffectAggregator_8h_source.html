195 /**
196 * Evaluates the result of the specified base value run through each existing evaluation channel's modifiers in numeric order
197 * with the specified evaluation parameters. The result of the evaluation of an individual channel acts as the new base value
198 * to the channel that follows it until all channels have been evaluated.
199 *
200 * EXAMPLE: Base Value: 2, Channel 0 has a +2 Additive Mod, Channel 1 is provided a base value of 4 to run through its modifiers
201 *
202 * @param InlineBaseValue Initial base value to use in the first evaluation channel
203 * @param Parameters Additional evaluation parameters
204 *
205 * @return Result of the specified base value run through each modifier in each evaluation channel in numeric order
206 */
207 UE_API float EvaluateWithBase(float InlineBaseValue, const FAggregatorEvaluateParameters& Parameters) const;
208
209 /**
210 * Similar to EvaluateWithBase (see comment there for details), but terminates early after evaluating the specified final channel instead of
211 * continuing through every possible channel
212 *
213 * @param InlineBaseValue Initial base value to use in the first evaluation channel
214 * @param Parameters Additional evaluation parameters
215 * @param FinalChannel Channel to terminate evaluation with (inclusive)
216 *
217 * @return Result of the specified base value run through each modifier in each evaluation channel in numeric order
218 */
219 UE_API float EvaluateWithBaseToChannel(float InlineBaseValue, const FAggregatorEvaluateParameters& Parameters, EGameplayModEvaluationChannel FinalChannel) const;
220
221 /**
222 * Evaluates a final value in reverse, attempting to determine a base value from the modifiers within all of the channels.
223 * The operation proceeds through all channels in reverse order, with the result of the evaluation of an individual channel used
224 * as the new final value to the channel that precedes it numerically. If any channel has a condition that prevents it from
225 * computing correctly (such as an override mod), this function just returns the original final value. This is predominantly used
226 * for filling in base values on clients from replication for float-based attributes.
227 *
228 * @note This will be deprecated/removed soon with the transition to struct-based attributes.
229 *
230 * @param FinalValue Final value to reverse evaluate
231 * @param Parameters Evaluation parameters to use for the reverse evaluation
232 *
233 * @return If possible, the base value from the reverse evaluation. If not possible, the original final value is returned.
234 */
