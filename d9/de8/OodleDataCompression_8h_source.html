11// OodleDataCompression :
12// Unreal API for direct access to Oodle Data lossless data compression
13// for manual encoding (not in Pak/iostore)
14// NOTE : for any data that will be stored to disk, you should not be compressing it yourself!
15// allow the pak/iostore system to choose the compressor!
16//
17// Let me emphasize again : for data to be stored in packages for shipping games, do NOT use this
18// let the pak/iostore system choose the compression from the config options and platform settings
19//
20// OodleDataCompression.h is for utility compression in non-shipping game package scenarios
21// eg. uassets, storage caches and back-ends, large network transfers
22//
23// For Compression to/from TArray and such higher level actions use OodleDataCompressionUtil.h

32 * this mostly trades decompression speed vs compression ratio
33 * encode speed is determined by ECompressionLevel , not the compressor choice.

35 * From fastest to slowest (to decode) : Selkie, Mermaid, Kraken, Leviathan

62/**
63 * ECompressionLevel : Choose the Oodle Compression Level
64 * this mostly trades encode speed vs compression ratio
65 * decode speed is determined by choice of compressor (ECompressor)
66 *
67 * If in doubt start with "Normal" (level 4) then move up or down from there
68 *
69 * The standard range is in "SuperFast" - "Normal" (levels 1-4)
70 *
71 * "HyperFast" levels are for real-time encoding with minimal compression
72 *
73 * The "Optimal" levels are much slower to encode, but provide more compression
74 * they are intended for offline cooks
75 *
76 * representative encode speeds with compression ratio trade off :
77
78Kraken-4 : 1.55:1 , 718.3 enc MB/s
79Kraken-3 : 1.71:1 , 541.8 enc MB/s
80Kraken-2 : 1.88:1 , 434.0 enc MB/s
81Kraken-1 : 2.10:1 , 369.1 enc MB/s
82Kraken1 : 2.27:1 , 242.6 enc MB/s
83Kraken2 : 2.32:1 , 157.4 enc MB/s
84Kraken3 : 2.39:1 , 34.8 enc MB/s
85Kraken4 : 2.44:1 , 22.4 enc MB/s
86Kraken5 : 2.55:1 , 10.1 enc MB/s
87Kraken6 : 2.56:1 , 5.4 enc MB/s
88Kraken7 : 2.64:1 , 3.7 enc MB/s
89
90 */

253/**
254* Decode compressed data that was made by Compress, using multiple threads when possible.
255*
256* UncompressedSize must match exactly the uncompressed size that was used at encode time. No partial decodes.
257*
258* There's no penalty to using this on small buffers, it will just decode synchronously on the calling thread in that case.
259* DecompressParallel can be used to decode data written by Compress or CompressParallel.
260* DecompressParallel can be used in all places you would call Decompress.
261*
262* DecompressParallel can only use more than 1 thread if the encoding was done with CompressIndependentChunks = true
263*
264* @param OutUncompressedData output buffer where uncompressed data is written
265* @param UncompressedSize number of bytes to decompress
