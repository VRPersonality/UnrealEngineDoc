26/** Used by the render graph builder to validate correct usage of the graph API from setup to execution.
27 * Validation is compiled out in shipping builds. This class tracks resources and passes as they are
28 * added to the graph. It will then validate execution of the graph, including whether resources are
29 * used during execution, and that they are properly produced before being consumed. All found issues
30 * must be clear enough to help the user identify the problem in client code. Validation should occur
31 * as soon as possible in the graph lifecycle. It's much easier to catch an issue at the setup location
32 * rather than during deferred execution.
33 *
34 * Finally, this class is designed for user validation, not for internal graph validation. In other words,
35 * if the user can break the graph externally via the client-facing API, this validation layer should catch it.
36 * Any internal validation of the graph state should be kept out of this class in order to provide a clear
37 * and modular location to extend the validation layer as well as clearly separate the graph implementation
38 * details from events in the graph.
39 */
