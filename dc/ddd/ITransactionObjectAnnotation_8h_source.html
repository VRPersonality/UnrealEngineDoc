10/**
11 * Interface for transaction object annotations.
12 *
13 * Transaction object annotations are used for attaching additional user defined data to a transaction.
14 * This is sometimes useful, because the transaction system only remembers changes that are serializable
15 * on the UObject that a modification was performed on, but it does not see other changes that may have
16 * to be remembered in order to properly restore the object internals.
17 */
