31/**
32 * At certain times we want a single menu entry that represents a set of
33 * UBlueprintNodeSpawners (generally when all those UBlueprintNodeSpawners wrap
34 * the same UField). We do this to keep the menu less jumbled, and instead
35 * use the drag/drop action to present a sub-menu to the user (so they can pick
36 * the the node type that they want). We do this with both delegates and
37 * variable nodes (where the user can pick a getter vs. a setter, etc.).
38 *
39 * This class represents those "consolidated" actions, and essentially serves
40 * as a FDragDropOperation spawner. It wraps a single UBlueprintNodeSpawner (any
41 * one of the set that it is supposed to represent), that it uses to determine
42 * the proper FDragDropOperation.
