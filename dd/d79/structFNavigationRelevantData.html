◆ bPendingLazyGeometryGathering

uint32 FNavigationRelevantData::bPendingLazyGeometryGathering

get set to true when lazy navigation exporting is enabled and this navigation data has "potential" of containing geometry data.

First access will result in gathering the data and setting this flag back to false. Mind that this flag can go back to 'true' if related data gets cleared out.

◆ ShouldUseGeometryDelegate

FFilterNavDataDelegate FNavigationRelevantData::ShouldUseGeometryDelegate

called to check if hosted geometry should be used for given FNavDataConfig.

If not set then "true" is assumed. Might want to set bUseVirtualGeometryFilteringAndDirtying to true in the Navmesh class you are excluding geometry from. This will improve cpu performance by stopping the navmesh from dirtying tiles requested by actors being excluded by this delegate.
