127 /**
128 * Start tracing to a target (network connection or file) with an active set of channels. If a connection is
129 * already active this call does nothing.
130 * @param Type Type of connection. Network or File type. If a custom target is desired use \ref FTraceAuxiliary::Relay.
131 * @param Target String to use for connection. See /ref EConnectionType for details.
132 * @param Channels Comma separated list of channels to enable. Default set of channels are enabled if argument is not specified. If the pointer is null no channels are enabled.
133 * @param Options Optional additional tracing options.
134 * @param LogCategory Log channel to output messages to. Default set to 'Core'.
135 * @return True when successfully starting the trace, false if the data connection could not be made.
136 */
137 static CORE_API bool Start(EConnectionType Type, const TCHAR* Target, const TCHAR* Channels = TEXT("default"), FOptions* Options = nullptr, const FLogCategoryAlias& LogCategory = LogCore);
138
139 /**
140 * Start tracing to a custom target by providing a handle and a write and close function. If a connection is already active this call does nothing.
141 * @param Handle Anonymous handle to trace to. This will be passed to the writer and close functions.
142 * @param WriteFunc A function to handle writing of trace data
143 * @param CloseFunc A function to handle closing of the data stream.
144 * @param Channels Comma separated list of channels to enable. Default set of channels are enabled if argument is not specified. If the pointer is null no channels are enabled.
145 * @param Options Optional additional tracing options.
146 * @return True when successfully starting the trace, false if the data connection could not be made.
147 */
148 static CORE_API bool Relay(UPTRINT Handle, UE::Trace::IoWriteFunc WriteFunc, UE::Trace::IoCloseFunc CloseFunc, const TCHAR* Channels = TEXT("default"), const FOptions* Options = nullptr);

289 /**
290 * Delegate that triggers when a connection is established. Gives subscribers a chance to trace events that appear
291 * after important events but before regular events (including tail). The following restrictions apply:
292 * * Only NoSync event types can be emitted.
293 * * Important events should not be emitted. They will appear after the events in the tail.
294 * * Callback is issued from a worker thread. User is responsible to synchronize shared resources.
295 *
296 * @note This is an advanced feature to avoid using important events in cases where event data can be recalled easily.
297 *
298 * @param Callback Delegate to call on new connections.
299 */
