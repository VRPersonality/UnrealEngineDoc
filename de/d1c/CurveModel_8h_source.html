249 /**
250 * Gets the value of ALL attributes including those that are auto-computed.
251 * Use this version if you want to know values e.g. the tangents or weights. Useful for UI code that visualizes tangents.
252 * The result is not intended to be passed to SetKeyAttributes; calling SetKeyAttributes would correct the passed in attributes to reflect the user settings.
253 *
254 * Typically, attributes reflect the settings that the user has manually configured for the keys, so certain attributes may remain unset.
255 * For instance, when TangentMode == RCTM_Auto, tangents and weights are automatically computed, meaning attributes like ArriveTangent are not explicitly set.
256 * Setting ArriveTangent would imply a user-defined value, which is incompatible with TangentMode == RCTM_Auto.
257 *
258 * In some situations, you don't want to know the values the user has set but the values as they would be used for evaluation, e.g. tangents,
259 * weights, etc. For these cases, you can use GetKeyAttributesIncludingAutoComputed.

474 /**
475 * This is an internal name used to try to match different curves with each other. When saving
476 * and later restoring curves on a different set of curves we need a name that gives enough context
477 * to match them up by intention, and not long or short name. For example, a curve might have a short
478 * name of "X", and a long name of "Floor.Transform.Location.X". If you wanted to copy a set of transform
479 * curves and paste them onto another transform, we use this context to match the names together to ensure
480 * your Transform.X gets applied to the other Transform.X - in this example the intention is
481 * for the curve to represent a "Location.X" (so it should be pasteable on any other curve which says
482 * their context is a "Location.X" as well). This is more reliable and more flexible than relying on
483 * short display names (not enough context in the case of seeing Location.X, and Scale.X) and better
484 * than relying on long display names (too much context and no reliable way to substring them).
