17 /**
18 * Easing function when zooming out on the X-axis using Alt+Shift+RMB + move mouse left/right.
19 *
20 * X-value is how far the mouse has travelled on the X-axis. Positive when zooming out, negative when zooming in.
21 * Y-Value is the resulting zoom multiplier. Must be >= 0.0. When zooming out (i.e. x < 0), the Y value must be <= 1.0 or zooming is unstable.
22 * Tip: You can use "CurveEditor.LogHorizontalZoomMultipliers" CVar to make tuning the x and y values easier.
23 *
24 * Example:
25 * - For Sequencer, the frame rate is the length of 1 (unit length).
26 * - So if Sequencer's target FPS is set to 30, and 45 frames are visible on the X-Axis in Curve Editor, we say the X-axis is 1.5 units wide.
27 * - We'd look up the value 1.5 in this easing function and use that as zoom multiplier.
28 */
29 UPROPERTY(config, EditAnywhere, Category="Curve Editor", meta=(
30 DisplayName="Right mouse (input axis)",

76 FCurveEditorZoomScaleConfig()
77 {
78 // By default, set this data up to do not additional scaling of the zoom.
79 HorizontalZoomScale.EditorCurveData.AddKey(0.0, 1.0);
80 HorizontalZoomScale.EditorCurveData.AddKey(1.0, 1.0); // So post infinity linear interpolation works
81 HorizontalZoomScale.EditorCurveData.DefaultValue = 1.0;
82 // While there is a limit for zooming in, for zooming out we want linear extrap so the user does not have to define an infinitely large X.
83 HorizontalZoomScale.EditorCurveData.PostInfinityExtrap = RCCE_Linear;
84
85 VerticalZoomScale.EditorCurveData.AddKey(0.0, 1.0);
86 VerticalZoomScale.EditorCurveData.AddKey(1.0, 1.0); // So post infinity linear interpolation works

88 // While there is a limit for zooming in, for zooming out we want linear extrap so the user does not have to define an infinitely large X.
89 VerticalZoomScale.EditorCurveData.PostInfinityExtrap = RCCE_Linear;
