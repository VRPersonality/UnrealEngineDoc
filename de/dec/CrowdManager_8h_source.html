41/**
42 * Crowd manager is responsible for handling crowds using Detour (Recast library)
43 *
44 * Agents will respect navmesh for all steering and avoidance updates,
45 * but it's slower than AvoidanceManager solution (RVO, cares only about agents)
46 *
47 * All agents will operate on the same navmesh data, which will be picked from
48 * navigation system defaults (UNavigationSystemV1::SupportedAgents[0])
49 *
50 * To use it, you have to add CrowdFollowingComponent to your agent
51 * (usually: replace class of PathFollowingComponent in AIController by adding
52 * those lines in controller's constructor
53 *
54 * ACrowdAIController::ACrowdAIController(const FObjectInitializer& ObjectInitializer)
55 * : Super(ObjectInitializer.SetDefaultSubobjectClass<UCrowdFollowingComponent>(TEXT("PathFollowingComponent")))
56 *
57 * or simply add both components and switch move requests between them)
58 *
59 * Actors that should be avoided, but are not being simulated by crowd (like players)
60 * should implement CrowdAgentInterface AND register/unregister themselves with crowd manager:
61 *
62 * UCrowdManager* CrowdManager = UCrowdManager::GetCurrent(this);
63 * if (CrowdManager)
64 * {
65 * CrowdManager->RegisterAgent(this);
66 * }
67 *
68 * Check flags in FCrowdDebug namespace (CrowdManager.cpp) for debugging options.
69 */
