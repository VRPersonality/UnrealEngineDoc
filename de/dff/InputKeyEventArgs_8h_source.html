15/**
16 * Input key event arguments. This is data that is required for the viewport client to
17 * process input via its InputKey/InputAxis functions.
18 *
19 * It represents a single "input event" which has happened which we want game or editor code
20 * to be able to process. This may accumulate several "raw" input events from the message
21 * handler into one with multiple sample readings, or it could be just one single key press event.
22 *
23 * This data can represent gamepad, keyboard, mouse, or touch data.
24 */
