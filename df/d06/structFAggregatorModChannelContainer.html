◆ EvaluateWithBase()

GAMEPLAYABILITIES_API float FAggregatorModChannelContainer::EvaluateWithBase ( float InlineBaseValue,
const FAggregatorEvaluateParameters & Parameters ) const

Evaluates the result of the specified base value run through each existing evaluation channel's modifiers in numeric order with the specified evaluation parameters.

The result of the evaluation of an individual channel acts as the new base value to the channel that follows it until all channels have been evaluated.

EXAMPLE: Base Value: 2, Channel 0 has a +2 Additive Mod, Channel 1 is provided a base value of 4 to run through its modifiers

Parameters
InlineBaseValueInitial base value to use in the first evaluation channel
ParametersAdditional evaluation parameters
Returns
Result of the specified base value run through each modifier in each evaluation channel in numeric order

◆ ReverseEvaluate()

GAMEPLAYABILITIES_API float FAggregatorModChannelContainer::ReverseEvaluate ( float FinalValue,
const FAggregatorEvaluateParameters & Parameters ) const

Evaluates a final value in reverse, attempting to determine a base value from the modifiers within all of the channels.

The operation proceeds through all channels in reverse order, with the result of the evaluation of an individual channel used as the new final value to the channel that precedes it numerically. If any channel has a condition that prevents it from computing correctly (such as an override mod), this function just returns the original final value. This is predominantly used for filling in base values on clients from replication for float-based attributes.

Note
This will be deprecated/removed soon with the transition to struct-based attributes.
Parameters
FinalValueFinal value to reverse evaluate
ParametersEvaluation parameters to use for the reverse evaluation
Returns
If possible, the base value from the reverse evaluation. If not possible, the original final value is returned.
