504/**
505 * UModelingObjectsCreationAPI is a base interface for functions that can be used to
506 * create various types of objects from Modeling Tools, or other sources. The "type" is
507 * very generic here - "Mesh", "Texture", etc - because this API is meant to provide
508 * an abstraction for Tools to emit different types of objects in different situations.
509 * For example an Tool might emit StaticMesh Asset/Actors in-Editor, but ProceduralMeshComponents at Runtime.
510 *
511 * The creation inputs are specified via the structs above (eg FCreateMeshObjectParams, FCreateTextureObjectParams),
512 * which are very extensive, kitchen-sink sort of structs. Generally "New Mesh Object"
513 * creation behavior will be very complex and so this API is really just a way to route
514 * the data, and very few guarantees can be made about any specific implementation.
515 *
516 * The assumed (but not really required) usage of instances of this type are that they
517 * will be registered with an InteractiveToolsContext's ContextObjectStore, and then
518 * fetched from there by Tools/Algorithms/etc that need to use these capabilities can
519 * use the UE::Modeling::CreateXObject() helper functions below. However the interface
520 * does not have any dependencies on this usage model.
521 *
522 * See UEditorModelingObjectsCreationAPI for an example implementation suitable for in-Editor use.
523 */
524UCLASS(MinimalAPI, Abstract, BlueprintType)
525class UModelingObjectsCreationAPI : public UObject
526{
527 GENERATED_BODY()
528public:
529 /**
530 * Create a new mesh object based on the data in CreateMeshParams
531 * @return a results data structure, containing a result code and information about any new objects created
532 */
533 UFUNCTION(BlueprintCallable, Category = "Modeling Objects")
534 virtual FCreateMeshObjectResult CreateMeshObject(const FCreateMeshObjectParams& CreateMeshParams) { return FCreateMeshObjectResult{ ECreateModelingObjectResult::Failed_Unknown }; }
535

543 /**
