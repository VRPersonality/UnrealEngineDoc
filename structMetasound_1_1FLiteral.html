Detailed Description

FLiteral represents a constant value in a Metasound graph and is primarily used to invoke the correct constructor of a Metasound data type.

To be constructed using a FLiteral, the Metasound data type must support a constructor which accepts one of the types supported by a FLiteral. The Metasound data type may optionally accept a const FOperatorSettings& in addition.

Example: // Somewhere before DECLARE_METASOUND_DATA_REFERENCE_TYPES(FAudioBuffer...), ParseFrom<FAudioBuffer>(int32) is defined // which means this data type can be created from an integer. Metasound::FAudioBuffer ::Metasound::ParseFrom<Metasound::FAudioBuffer>(int32 InNumFrames, const ::Metasound::FOperatorSettings&) { return Metasound::FAudioBuffer(InNumFrames); }

//... // In the frontend, we know that int and pass it to somewhere in MetasoundGraphCore... FLiteral InitParam(512);

//... // In the backend, we can safely construct an FAudioBuffer. if(InitParam.IsCompatibleWithType<Metasound::FAudioBuffer>()) { Metasound::FAudioBuffer AudioBuffer = InitParam.ParseTo<Metasound::FAudioBuffer>(); //... }
