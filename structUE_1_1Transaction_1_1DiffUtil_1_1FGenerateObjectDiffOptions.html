◆ bDiffDataEvenIfPendingKill

bool UE::Transaction::DiffUtil::FGenerateObjectDiffOptions::bDiffDataEvenIfPendingKill = false

Should we diff the object state even if it's considered pending kill?

When false: If the old object was pending kill and the new object isn't, then the diff will be performed against the archetype of the new object. If the old object wasn't pending kill and the new object is, then the data diff will be skipped and only the object info diffed.
